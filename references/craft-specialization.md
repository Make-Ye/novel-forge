# 题材专项技法

> **权威声明**: 本文件是类型专精（仙侠体系、现实主义、Save the Cat 结构）的权威参考。
> 战斗场景描写见 combat-scenes.md。

## 核心理念

**不同题材需要不同的专业技法库。** 修仙小说需要功法体系设计能力，现实主义需要陌生化技法，所有题材都需要结构模板。本文件整合三种题材专项技法模块。

---

## 一、武侠/修仙七模块系统

### 模块1：功法体系（5层设计）

```
第1层：基础概念
- 功法的存在形式（传承？自创？天赋觉醒？）
- 修炼的基本原理（灵气？真气？道韵？）
- 功法与境界的关系

第2层：分类框架
- 按属性分（金木水火土 / 阴阳 / 五行 / 特殊属性）
- 按品级分（黄级→玄级→地级→天级→仙级→神级）
- 按来源分（上古传承/宗门秘传/自创/天赐）

第3层：核心规则（必须明确且不自相矛盾）
- 修炼限制（根骨要求/天赋限制/时间成本）
- 突破条件（灵石？天材地宝？顿悟？渡劫？）
- 不可逆规则（散功重修的代价、走火入魔的后果）

第4层：差异化设计
- 同品级功法的区别（不是简单的威力大小）
- 功法的"性格"（刚猛/阴柔/变化/守成）
- 修炼者的个人烙印（同一功法不同人练出不同效果）

第5层：剧情接口
- 功法如何制造冲突（功法争夺、修炼瓶颈、功法克制）
- 功法如何推动剧情（新功法=新能力=新冒险）
- 功法如何服务主题（修炼的意义是什么？力量为了什么？）
```

### 模块2：外功体系（7类）

| 类型 | 特征 | 写法要点 |
|------|------|---------|
| 剑术 | 快、准、灵 | 招式名要有画面感，描写聚焦剑光轨迹 |
| 刀法 | 猛、霸、势 | 描写力量感和气势，每一刀有开山之威 |
| 拳掌 | 近身、刚猛/柔化 | 身体接触的质感，拳风掌力 |
| 暗器 | 隐蔽、精准、防不胜防 | 描写角度和时机，不是暗器本身 |
| 阵法 | 空间控制、配合 | 阵法效果的可视化描写 |
| 身法 | 速度、闪避、位移 | 环境与身法的配合（借力/地形） |
| 特殊 | 毒/蛊/咒/符/音 | 每种有独特的施展方式和限制条件 |

### 模块3：轻功/移动体系（5种）

```
1. 基础轻功：跳跃强化、壁虎游墙
2. 飞行术：御剑/御风/御物飞行
3. 瞬移类：短距离瞬移（限制条件：需要标记/冷却时间/灵力消耗）
4. 空间术：空间折叠/传送阵（高阶，必须有限制）
5. 特殊移动：潜行/隐身/穿墙（各有严格限制）
```

### 模块4：门派/势力设计

```
门派设计必须回答：
- 创派历史（谁创的？为什么？什么时候？）
- 核心理念（不只是"修仙"，要有一句话能说清的独特理念）
- 势力范围（地理、人口、资源、盟友）
- 内部结构（长老制？长老会？掌门独裁？弟子分级？）
- 与其他势力的关系（联盟/敌对/中立/暗中操控）
- 弱点和隐患（没有完美的势力）

禁忌：不要创建没有矛盾的门派。每个门派内部至少有一组对立势力。
```

### 模块5：战斗桥段库

| 桥段类型 | 描述 | 使用频率 |
|---------|------|---------|
| 暗杀/偷袭 | 不宣而战的突袭 | 中等 |
| 比武/切磋 | 正式规则下的战斗 | 高（前期） |
| 围攻/群战 | 一人对多人或多方混战 | 中等 |
| 追逃 | 一追一逃，紧张感主导 | 中等 |
| 阵法对决 | 两个阵法师/阵法的对抗 | 低（中后期） |
| 心理战 | 不直接交手，以势/言语/布局压制 | 中等 |
| 暴走/失控 | 角色失去理性，力量暴走 | 低（关键转折） |

### 模块6：兵器/法宝体系

```
兵器分级：
凡兵 → 法器 → 灵器 → 宝器 → 仙器 → 道器

每个等级需要定义：
- 获取难度
- 能力范围
- 与主人的绑定方式（认主？血炼？器灵？）
- 升级路径（可以成长吗？怎么成长？）
- 特殊能力（不只是攻击力数值，要有独特效果）

器灵设计规则：
- 器灵有独立人格，不是工具人
- 器灵与主人的关系有发展空间
- 器灵有自己的过去和执念
```

### 模块7：等级/境界体系

```
设计原则：
1. 每个境界有可感知的区别（不只是名字变了）
2. 境界之间的差距有"质感"（低境界看高境界是什么感觉）
3. 突破有明确的标志（外在表现/内在感受/他人认知）
4. 越高境界越难突破，但突破后的质变越大
5. 同境界内的个体差异（不是所有同境界的人一样强）

常见结构（可变体）：
入门 → 筑基 → 金丹 → 元婴 → 化神 → 合体 → 大乘 → 渡劫 → 仙人

自定义时的检查清单：
[ ] 境界名称有内在逻辑（不是随便取的）
[ ] 每个境界有独特特征（视觉/感知/能力标志）
[ ] 突破条件清晰且前后一致
[ ] 境界差距有画面感（不是"强了十倍"这种数字）
```

## 二、现实映射六模块系统

用于批判现实主义、历史映射、社会题材等作品。

### 模块1：批判现实主义技法

```
核心理念：不直接说教，通过故事让读者自己得出结论

技法1：典型化
- 角色是"典型"而非"特例"——代表一类人
- 事件是"常见"而非"罕见"——读者能代入
- 矛盾是"结构性"而非"个人化"——是制度问题不是坏人问题

技法2：细节真实
- 用真实的职业/行业/流程细节建立可信度
- 读者看完觉得"这就是真实发生的事"
- 需要调研，不能凭想象

技法3：无声控诉
- 不加评论地展示不公
- 让角色在体制面前无力
- 读者的愤怒来自"这确实会发生"的共鸣

技法4：多声部叙事
- 不同阶层/立场的角色各说各话
- 没有一个角色代表"作者观点"
- 读者自行判断
```

### 模块2：陌生化10技法

让熟悉的事物变得陌生，重新被"看见"。

| 技法 | 描述 | 示例 |
|------|------|------|
| 外来者视角 | 用陌生人的眼睛看日常 | 一个古代人看手机："发光的板子，所有人的魂都被吸进去了" |
| 尺度变化 | 放大或缩小观察尺度 | 把一粒灰尘描述成一座山 |
| 时间错位 | 用另一个时代的标准看现在 | "他们把人关在发光的盒子里八小时，管这叫'上班'" |
| 物化 | 把人当物品描述 | "他们被码放在工位上，像货架上的商品，每一件都有自己的条码" |
| 去名词化 | 不用常见名称，只描述功能 | 不说"手机"——"那个所有人都低头看的东西" |
| 对比并置 | 把两件不相干的事放在一起 | 写豪华宴会的菜名，紧接着写食堂的剩菜 |
| 儿童视角 | 用孩子的逻辑理解成人世界 | "爸爸每天都去一个让他不开心的地方，但不去就会更不开心" |
| 动物视角 | 用动物的眼睛看人类行为 | "两足兽每天早上把自己塞进铁盒子里，然后在另一个铁盒子里待一天" |
| 夸张到荒诞 | 把现实推到极端 | 每天通勤四小时的人，一生在地铁里度过的时间相当于坐牢五年 |
| 拼贴/蒙太奇 | 不同场景碎片拼接 | 办公室PPT+工地搬砖+夜市摆摊——同一个晚上的三个人 |

### 模块3：荣格原型应用

| 原型 | 功能 | 在小说中的角色 |
|------|------|--------------|
| 英雄 | 推进故事、承担责任 | 主角 |
| 阴影 | 压抑的、被否认的自我 | 反派或主角的暗面 |
| 阿尼玛/阿尼姆斯 | 异性面的自我 | 恋人/精神导师 |
| 智者 | 提供知识和指引 | 师父/导师角色 |
| 骗子/捣蛋鬼 | 打破秩序、制造变化 | 亦正亦邪的角色 |
| 母亲 | 养育、保护（也可能窒息） | 母性角色/组织 |
| 父亲 | 秩序、权威（也可能压迫） | 父性角色/体制 |

**使用规则：**
- 每个角色至少对应一个原型，但可以有多个层面的原型
- 反派不应该是纯粹的"阴影"——要有自己的逻辑和合理性
- 原型是参考，不是枷锁——可以混合、颠覆、重新诠释

### 模块4：社会映射模板

```
映射规则：
1. 小说中的世界/组织/制度 → 对应现实中的什么
2. 这种映射是否清晰但不露骨
3. 读者能否自己发现映射（而不是被按头说教）
4. 映射是否有深度（不是简单的一一对应）

映射维度：
- 权力结构（谁在掌权？权力如何分配？如何维持？）
- 资源分配（谁拥有？谁缺乏？如何流动？）
- 阶层固化（有上升通道吗？代价是什么？）
- 信息控制（谁知道什么？谁被蒙在鼓里？）
- 暴力垄断（谁掌握武力？如何使用？）
```

### 模块5：法律与合规边界

```
当小说涉及现实社会议题时的自我审查：

可以做：
- 用虚构世界映射真实问题
- 展示社会矛盾但不煽动
- 角色有不同政治立场
- 讨论历史事件的不同解读

不能做：
- 直接影射真实政治人物
- 宣扬特定政治立场
- 描写具体的暴力抗争方法
- 歧视特定群体

灰色地带处理：
- 遇到不确定的内容，先问用户
- 宁可含蓄，不要直白
- 用"如果...会怎样"的假设框架替代直接评论
```

### 模块6：素材搜索与处理

```
为现实题材小说收集素材的方法：

1. 田野调查式：去真实场景观察（市场、工地、办公室、医院）
2. 访谈式：和真实职业的人聊天（外卖员、护士、程序员、老师）
3. 文献式：阅读相关报道、纪录片、行业报告
4. 体验式：亲自尝试角色会做的事
5. 数据式：查找统计数据支撑细节

素材处理三原则：
- 不照搬真人真事（保护隐私，避免侵权）
- 抓住"质感"而非"事实"（不一定非要真实事件，但必须有真实的细节质感）
- 用一个细节代表一类经验（不是堆砌所有素材）
```

## 三、Save the Cat! 节拍表（三层适配）

### 原始15拍

| # | 节拍 | 占比 | 功能 |
|---|------|------|------|
| 1 | Opening Image | 1% | 开场画面——主角的"之前"状态 |
| 2 | Theme Stated | 5% | 主题暗示——故事要讨论什么 |
| 3 | Set-Up | 1-10% | 建置——主角的世界和缺陷 |
| 4 | Catalyst | 10% | 催化剂——打破日常的事件 |
| 5 | Debate | 10-20% | 犹豫——主角是否要行动 |
| 6 | Break into Two | 20% | 进入第二幕——主角做出选择 |
| 7 | B Story | 22% | 副线——通常是关系线 |
| 8 | Fun and Games | 20-50% | 游戏时间——承诺的有趣内容 |
| 9 | Midpoint | 50% | 中点——虚假胜利或虚假失败 |
| 10 | Bad Guys Close In | 50-75% | 敌人逼近——困境升级 |
| 11 | All Is Lost | 75% | 一切尽失——最低谷 |
| 12 | Dark Night of Soul | 75-80% | 灵魂暗夜——反思和转变 |
| 13 | Break into Three | 80% | 进入第三幕——重新振作 |
| 14 | Finale | 80-99% | 终局——用所学到的解决问题 |
| 15 | Final Image | 99% | 终场画面——主角的"之后"状态 |

### 三层适配

**短片/短篇（1-5章）：** 15拍压缩到5个关键节点
```
Opening + Set-Up → Catalyst + Debate → Fun&Games + Midpoint → All Is Lost → Finale
```

**中篇（10-30章）：** 完整15拍，每拍对应1-2章
```
按比例分配章节数即可。
例：20章小说，10%=第2章是催化剂，20%=第4章进入第二幕，50%=第10章中点
```

**长篇连载（100+章）：** 每个大弧线（卷）是一套完整的15拍
```
第1卷：用15拍建立世界和主角
第2-N卷：每卷有自己完整的15拍弧线
最终卷：回归全局15拍的结构终点

注意：长篇中"Fun and Games"是最长的部分，需要拆分成多个小弧线
```

### 10种故事原型

| 原型 | 核心问题 | 经典结构 |
|------|---------|---------|
| 怪物在屋内 | 主角和威胁被困在同一空间 | 密闭空间+逐步揭露+最终对决 |
| 金羊毛 | 主角踏上旅途寻找某物 | 旅途+伙伴+障碍+发现真正的宝物不是目标 |
| 出局愿望 | 主角许愿成真但付出代价 | 愿望→享受→代价→教训→撤销或接受 |
| 伙伴情 | 两个不搭的人必须合作 | 冲突→磨合→默契→危机→最终的信任 |
| 飞屋环游记 | 日常生活被打破的冒险 | 无聊日常→被迫冒险→发现自己→回归但已不同 |
| 复仇 | 主角被伤害后寻求报复 | 伤害→计划→执行→复仇的空虚→新的选择 |
| 解谜 | 主角揭开一个谜团 | 谜题→线索→误导→真相→更大的谜题 |
| 痴汉喜剧 | 主角的缺点被放大成喜剧 | 缺陷暴露→尴尬升级→被迫面对→成长 |
| 傻人有傻福 | 天真的人在复杂世界中 | 天真进入→被低估→用纯真解决问题 |
| 灰姑娘 | 被压制的人终获认可 | 被压→机遇→被夺→真相揭露→正名 |

## 四、场景设计：主动场景与被动场景

### 主动场景（Action Scene）

```
结构：Goal → Conflict → Disaster
- Goal：角色进入场景时的目标
- Conflict：遇到阻碍
- Disaster：结果比预期更糟（或转向新方向）

适用：推进剧情、制造转折
```

### 被动场景（Reaction Scene）

```
结构：Reaction → Dilemma → Decision
- Reaction：对上一个灾难的情绪反应
- Dilemma：面临两难选择
- Decision：做出决定（成为下一个主动场景的Goal）

适用：角色深化、消化前情、蓄力
```

### 交替规则

```
主动→被动→主动→被动 是最基本的节奏。
但不是死板的——可以：
- 连续两个主动场景（加速节奏）
- 延长的被动场景（蓄力效果）
- 在主动场景中嵌入微型被动段落

禁忌：连续三个以上的主动场景——读者会疲劳。
禁忌：连续两个以上的完整被动场景——节奏太慢。
```
