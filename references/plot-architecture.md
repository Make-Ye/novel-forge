# Plot Architecture: 情节架构参考

> **权威声明**: 本文件是情节架构的权威参考。
> 大纲模板见 outline-template-guide.md。子剧情管理见 subplot-management.md。

> **AUTHORITATIVE REFERENCE** — 本文件是情节结构的权威参考源。核心问题定义、网文流派结构、选择树、FAIL/PASS标准均以本文件为准。张力曲线的视觉化和节奏控制由 pacing-curve-reference 负责，不在此重复。

## 核心理念

Plot is THEME made physical. 每一个情节点都是关于主题的一次论证。先从主题出发，再设计结构，最后填充事件——这个顺序不能反。

---

## 一、从主题到情节

### 主题先行对话

在讨论任何情节事件之前，先确立主题核心。

**开场问题：**
- "如果你的故事结束后，读者只记住一句话，你希望那句话是什么？"
- "你最初想写的那个念头是什么？不是情节，是那个让你觉得'我一定要写这个'的感觉。"
- "主角到故事结束时，会知道一个他在开始时不知道的什么事？"

**主题 = 故事通过行动做出的论证。**

### 主题到情节的桥接

主题确立后，将它与结构对接：

```
Theme: "真正的自由是接受约束"

这意味着：
- 主角必须一开始持有反面信念（自由 = 不受任何约束）
- 情节必须通过升级的挑战来测试这个信念
- 中点必须让旧信念出现裂痕
- 高潮必须逼迫主角做出反映主题的选择
- 结局必须用行动展示主题真相
```

## 二、中央问题：让AI能生成具体情节的模板

**旧定义（太抽象）：** "故事回答的那个问题。"
**新定义：** 一个填充了具体变量的句子，任何看到它的人（包括AI）都能立刻生成方向一致的情节。

### 中央问题模板

```
"[主角名] 能否在 [期限/条件] 之前，克服 [核心障碍]，实现 [核心目标]？"
```

### 具体填法与示例

| 类型 | 填法 | 示例 |
|------|------|------|
| 升级流 | 障碍=境界瓶颈, 目标=突破, 条件=资源耗尽前 | "林墨能否在天才大选之前，突破练气期的瓶颈，拿到进入秘境的资格？" |
| 系统流 | 障碍=任务时限, 目标=奖励/升级, 条件=惩罚触发前 | "苏晨能否在系统倒计时结束前，完成'征服三家商会'的任务，避免属性扣除？" |
| 重生流 | 障碍=蝴蝶效应, 目标=改变命运, 条件=关键节点前 | "顾远能否在上一世的灾难重演之前，利用前世记忆阻止那场矿难？" |
| 打脸流 | 障碍=社会偏见, 目标=证明自己, 条件=公审/对决前 | "叶尘能否在宗门大比上，证明自己不是废物，让所有嘲笑他的人闭嘴？" |
| 情感 | 障碍=内心创伤, 目标=关系, 条件=对方离开前 | "沈默能否在江晚离开这座城市之前，克服十年前的愧疚，说出真话？" |
| 权谋 | 障碍=阴谋, 目标=夺权/自保, 条件=暴露/失势前 | "顾青能否在朝会之前，识破三皇子的连环计，保住父亲的兵权？" |

### 中央问题的检验标准

一个好的中央问题必须满足：
- [ ] 拿给一个陌生人，他能猜出故事的大致走向（具体性）
- [ ] 读者看到一半时，脑子里会浮现这个问题（贯穿性）
- [ ] 高潮时这个问题得到回答（完整性）
- [ ] 答案不是显而易见的（悬念性）

## 三、经典结构框架（压缩版）

以下结构在 pacing-curve-reference 中有详细的张力曲线图。此处仅列出核心要点和适用判断。

### 3-Act（三幕结构）

| 幕 | 功能 | 占比 | 网文适配 |
|----|------|------|----------|
| Act 1 | 建置 | 25% | 一个小弧线（3-5章） |
| Act 2 | 对抗 | 50% | 2-3个小弧线，中点需有转折 |
| Act 3 | 解决 | 25% | 最终弧线，高潮+余波 |

**适合：** 20万字以内、单主角、目标明确的故事。缺点是第二幕容易塌。

### 4-Act（四幕结构）

将第二幕从"反应"和"主动出击"中间劈开，中点成为结构支柱。

**适合：** 20-50万字、有明确中点翻转的故事。解决了三幕的中段塌陷。

### 5-Act（五幕结构）

在四幕基础上再加一层：展开->上升->危机（高潮）->下降->解决。

**适合：** 50万字以上、多线叙事、群像、史诗体量。复杂度最高，需配合小弧线使用。

### 结构选择原则

1. 体量决定大结构（短篇用3-act，长篇用4/5-act）
2. **大结构套小弧线**是网文铁律——即使整体是5-act，每一幕内部仍按3-5章的小弧线运行
3. 小弧线的张力曲线由 pacing-curve-reference 管理

## 四、网文专属结构模式

### 4.1 升级流（Upgrade Flow）

**核心驱动力：** 力量等级的阶梯式上升。读者为"变强"本身而阅读。

#### 结构节拍

```
[初始弱小] → [发现修炼路径] → [首次突破（小爽）] → [新环境/新对手]
→ [瓶颈期] → [瓶颈突破（大爽）] → [境界跃迁] → [新世界打开] → 循环
```

#### 小境界/大境界节奏

这是升级流的命脉。区分两种层次：

| 层次 | 特征 | 频率 | 读者反应 |
|------|------|------|----------|
| 小境界突破 | 属性/技能的小幅提升 | 每10-20章一次 | "稳了，在进步" |
| 大境界跃迁 | 质变，进入全新层级/世界 | 每50-100章一次 | "卧槽，世界变了" |

**节奏模式：**
```
小境界1 → 小境界2 → 小境界3 → [瓶颈] → 大境界跃迁
   ↑                                      |
   └────── 新的小境界循环 ←───────────────┘
```

**关键规则：**
- 每次大境界跃迁必须伴随**场景/地图/社会关系的全面刷新**——不能只是数值变化
- 瓶颈期不能太长（不超过15-20章），否则读者会觉得"卡住了"
- 小境界突破之间必须有事件驱动（不是纯修炼），每个突破对应一场挑战或冲突

#### AI生成提示

当使用升级流时，大纲生成需要明确：
- 当前境界体系（至少列出当前大境界和下一个大境界）
- 下一个突破点在哪几章
- 突破的触发条件（不是"修炼够了"，而是"在某场战斗中领悟了X"）

---

### 4.2 系统流（System Flow）

**核心驱动力：** 任务-奖励循环。读者为"系统界面弹出新任务/新奖励"的多巴胺而阅读。

#### 结构节拍

```
[系统激活] → [新手任务+首次奖励] → [奖励利用/能力展现（爽点）]
→ [任务升级] → [资源积累期] → [高难度任务+意外惩罚]
→ [系统进化/隐藏功能解锁] → [终极任务暗示] → [系统真相揭露]
```

#### 任务节奏模式

| 任务类型 | 频率 | 功能 | 示例 |
|----------|------|------|------|
| 日常任务 | 几乎每章 | 基础节奏，小额奖励 | "完成30分钟修炼：经验+100" |
| 周常任务 | 每5-10章 | 小节点，推动剧情 | "在一周内击败3名同级对手" |
| 月常/阶段任务 | 每20-40章 | 大节点，改变格局 | "建立自己的势力" |
| 隐藏任务 | 不定期 | 惊喜感，打破节奏 | "触发隐藏条件：???" |
| 终极任务 | 后期 | 主线高潮 | "系统的最终目的是……" |

**关键规则：**
- 系统流的核心矛盾：系统是工具还是主人？主角是利用系统还是被系统利用？
- 任务难度必须阶梯式上升，但不能让读者觉得"系统在故意刁难"——每次难度提升要有叙事上的理由
- 奖励不能只给数值——最好的奖励是**新的叙事可能性**（解锁新地图、新人物、新能力类型）
- 系统必须有"不完美"之处（冷却时间、代价、限制），否则没有戏剧张力

#### AI生成提示

当使用系统流时，大纲生成需要明确：
- 系统的核心规则和限制（不能全能）
- 当前阶段的任务难度等级
- 下一次系统进化/隐藏功能解锁的时机
- 系统与主线冲突的关系（系统任务是否与主角的真实目标一致？）

---

### 4.3 重生流（Rebirth Flow）

**核心驱动力：** 信息不对称。读者和主角共享"未来知识"，享受预知带来的碾压感和蝴蝶效应带来的意外感。

#### 结构节拍

```
[死亡/回归] → [确认时间线/盘点已知信息] → [首次利用前世记忆（爽点）]
→ [蝴蝶效应初现] → [已知事件开始偏移] → [关键分歧点]
→ [时间线彻底改变/新危机出现] → [前世记忆失效] → [靠自身能力解决终局]
```

#### 已知/未知/偏移 三层张力

```
         读者的认知
           ↑
已知区 ----→  主角知道会发生什么，读者也知道
           ↓     张力来自"他能不能利用好"
未知区 --------→ 主角不知道的事（前世也发生了但他没注意到）
           ↑     张力来自"有他遗漏的信息"
偏移区 --------→ 因为蝴蝶效应，事情已经和前世不一样了
                 张力来自"前世的记忆开始不可靠了"
```

**节奏模式：** 故事前30%以"已知区"为主（利用前世知识的爽感），30-60%逐渐进入"偏移区"（意外不断），60%以后以"未知区"为主（前世记忆失效，主角必须靠现在的能力）。

**关键规则：**
- 主角不能记住前世的所有细节——必须有信息盲区
- 蝴蝶效应必须**逐步放大**，不能第一次行动就让时间线面目全非
- 最佳设计：主角试图阻止的事件A，因为他的干预反而以更严重的形式发生了
- 终局必须让主角在没有前世记忆可依赖的情况下解决问题——这是角色成长的终极证明

#### AI生成提示

当使用重生流时，大纲生成需要明确：
- 主角前世记忆的时间范围和精度（他记得什么、不记得什么）
- 当前处于已知区/偏移区/未知区的哪个阶段
- 下一个蝴蝶效应触发点
- 已偏移的事件列表（追踪时间线变化）

---

### 4.4 打脸流（Face-Slapping Flow）

**核心驱动力：** 被低估者证明自己的快感。读者为"让看不起我的人后悔"而阅读。

#### 结构节拍

```
[屈辱/被低估] → [隐藏优势初显端倪] → [首次小规模打脸（爽点）]
→ [新的质疑/更高的对手] → [蓄力期（对方嚣张升级）]
→ [更大规模的打脸] → [终极对手出场] → [终极大打脸]
```

#### 压抑-释放节奏

打脸流的核心不是"打脸"本身，而是打脸之前的**压抑积累**：

```
压抑强度
  10 |                              ★ 终极打脸
   9 |                         ╱
   8 |              ★ 大打脸 ╱
   7 |            ╱     ╲  ╱
   6 |     ╱╲   ╱        ╳
   5 |   ╱   ╲ ╱      ╱    ╲
   4 |  ╱     ╳    ╱         ╲
   3 | ╱      ╲  ╱             ╲
   2 |╱   小打脸╲                ╲
   1 |           ╲压抑            ╲  ← 此处读者最想打人
     +-----------------------------------
     屈辱  蓄力 小释放 蓄力 大释放 终极蓄力 终极释放
```

**关键规则：**
- 每次打脸之前必须有**至少等量的压抑**——压抑越深，释放越爽
- 对手必须**递进式升级**：小人物 → 中层势力 → 顶级势力。不能打完大佬又回去打小喽啰
- 主角的"隐藏优势"必须有合理解释，不能是纯运气
- 最好的打脸不是主角自己说的，而是**旁观者的反应**——"他竟然……怎么可能！"
- 打脸不能无止境——每个大打脸之后需要有一段"被认可"的温情期，让读者喘息

#### AI生成提示

当使用打脸流时，大纲生成需要明确：
- 当前压抑等级和对应的对手层级
- 下一次打脸的触发场景（公开场合 > 私下）
- 打脸的旁观者是谁（旁观者越重要，爽感越强）

## 五、FAIL / PASS 情节标准

### FAIL 情节（必须避免）

| FAIL特征 | 说明 | 典型表现 |
|----------|------|----------|
| 事件被动 | 事情发生在主角身上，主角没有主动选择 | "主角被追杀所以逃跑"——这是反应，不是选择 |
| 无代价解决 | 问题被外力解决，主角没有付出 | 天降救兵、金手指自动激活、巧合 |
| 无选择的转折 | 转折点由信息揭露驱动，而非主角的决策 | "原来反派是主角的父亲"——这只是一个事实，不是一个选择 |
| 线性升级 | 主角只变得更强，没有真正的挫折或道德困境 | 每场战斗都赢，每次选择都正确 |
| 牺牲缺失 | 高潮不需要主角放弃任何重要的东西 | 打败了反派，什么都没失去 |

### PASS 情节（必须达到）

| PASS特征 | 说明 | 如何实现 |
|----------|------|----------|
| 每个转折 = 一个选择 | 转折点不是"发生了什么"，而是"主角选择了什么" | 在每个关键节点设计两个都不理想但必须选的选项 |
| 代价可见 | 每次前进都有失去 | 升级流：突破消耗寿元/重要关系；打脸流：打脸的代价是更大的敌人 |
| 信息 + 选择 | 信息揭露的意义在于它改变了选择，而非仅仅改变认知 | "反派是主角的父亲" → 主角现在必须选择"杀还是不杀" |
| 挫折有因 | 失败不是随机的，而是主角的缺陷导致的 | 傲慢导致轻敌，信任导致被背叛，保护欲导致错失良机 |
| 高潮 = 牺牲 | 最终胜利必须以某种牺牲为代价 | 牺牲可以是：能力、关系、信念、甚至"过去的自己" |

### 选择设计模板

每个关键转折点使用此模板检验：

```
转折点 #[编号]：[简要描述]

主角面临的选择：
  选项A：[描述] → 后果：[描述]（看起来更安全但违背主题）
  选项B：[描述] → 后果：[描述]（看起来更危险但符合主题）

他选了：[A/B]
代价是：[他失去了什么]
这个选择说明了他：[角色特征的变化]
```

**示例：**
```
转折点 #3：发现师妹是敌方的卧底

选项A：假装不知道，继续利用她获取敌方信息 → 后果：背叛自己的感情，变成自己讨厌的人
选项B：当面揭穿，给她一个解释的机会 → 后果：可能被反杀，失去战略优势

他选了：B
代价是：失去了唯一掌握敌方情报的渠道，被师父责罚
这个选择说明了他：从"只看结果"变成了"过程也要对得起自己"
```

## 六、结构选择决策树

根据题材和流派，推荐首选和备选结构组合。

### 决策流程

```
你写的是什么类型？
│
├─ 玄幻/仙侠（修真升级为核心）
│  ├─ 主线 = 升级流
│  ├─ 副线 = 打脸流（对手递进）
│  ├─ 大结构 = 5-act（长篇）或 4-act（中篇）
│  └─ 小弧线 = 每个小境界一个小弧线
│
├─ 都市系统/游戏系统
│  ├─ 主线 = 系统流
│  ├─ 副线 = 升级流（系统提供的等级体系）
│  ├─ 大结构 = 4-act（中点=系统进化）
│  └─ 小弧线 = 每个阶段任务一个小弧线
│
├─ 重生/穿越（信息优势为核心）
│  ├─ 主线 = 重生流
│  ├─ 副线 = 升级流（利用信息快速变强）
│  ├─ 大结构 = 4-act（中点=时间线开始偏移）
│  └─ 小弧线 = 每个前世事件一个小弧线
│
├─ 都市/商战/校园（无超能力）
│  ├─ 主线 = 打脸流
│  ├─ 副线 = 情感线（人物关系深化）
│  ├─ 大结构 = 3-act（紧凑）或 4-act
│  └─ 小弧线 = 每个对手一个小弧线
│
├─ 悬疑/推理/解谜
│  ├─ 主线 = 信息揭露驱动
│  ├─ 副线 = 角色关系谜团
│  ├─ 大结构 = 4-act（中点=最大反转）
│  └─ 小弧线 = 每个线索/嫌疑人一个小弧线
│
├─ 群像/史诗/权谋
│  ├─ 主线 = 多线并行，无单一主角
│  ├─ 副线 = 各线交叉互为因果
│  ├─ 大结构 = 5-act
│  └─ 小弧线 = 每个阵营/角色轮转
│
└─ 混合型（最常见的网文）
   ├─ 从上述选一个主线
   ├─ 从剩余中选1-2个副线
   ├─ 大结构按体量选（短=3act，中=4act，长=5act）
   └─ 小弧线按主线的节拍节奏运行
```

### 结构组合禁忌

| 禁忌 | 原因 |
|------|------|
| 升级流 + 系统流同时做主线 | 两个升级体系会打架，读者不知道该关注哪个 |
| 重生流 + 打脸流但无蝴蝶效应 | 利用前世知识打脸太容易，没有戏剧张力 |
| 纯打脸流超过200万字 | 压抑-释放模式在重复太多次后失效，读者会麻木 |
| 系统流但系统无限制/无代价 | 系统万能 = 没有冲突 = 没有故事 |
| 升级流但无社会关系刷新 | 纯数值变强无聊，每次境界跃迁必须换地图/换社交圈 |

---

## 七、关键架构决策

### 7.1 核心冲突设计

每个故事需要双层冲突：

**外部冲突：** 看得见的、有形的对手或问题。驱动情节事件。
**内部冲突：** 看不见的、情感性的缺陷或恐惧。驱动角色选择。

**最佳设计：** 外部冲突是内部冲突的物理具象化。打外在敌人的过程就是在面对内在的敌人。

**检验方法：** 如果把外部冲突换一个完全不同的形式（比如把"打败魔族"换成"打赢商战"），内部冲突是否仍然成立？如果是，说明两层冲突没有真正连接。

### 7.2 引发事件的位置

| 位置 | 效果 | 风险 | 适合 |
|------|------|------|------|
| 第1章 | 即时钩子，读者立刻进入 | 世界观铺垫不够 | 快节奏网文 |
| 第2-3章 | 简要建置后打破 | 建置可能拖沓 | 需要一点世界观的作品 |
| 第5章+ | 慢热蓄力 | 网文读者可能已经弃书 | 仅当早期章节有其他强吸引力 |

**网文铁律：** 引发事件不晚于第3章。如果在第3章之前没有出现改变主角日常的事件，读者流失率会急剧上升。

### 7.3 中点转折

中点是仅次于高潮的结构支柱。常见类型：

- **真相揭露：** 一个重大秘密或误解被揭开
- **角色反转：** 主角从被动反应变为主动出击
- **赌注升级：** 原来在赌的东西变大了、变私人的、变紧迫了
- **虚假胜利/失败：** 看似赢了/输了，其实恰好相反
- **镜像时刻：** 主角看到自己可能变成的样子（好或坏）

### 7.4 高潮设计

高潮必须同时解决外部和内部冲突。设计问题：

- "高潮是'最终决战'式、'最终选择'式，还是先选后战的组合？"
- "高潮时主角展现的'学到了什么'和开头的他有什么本质区别？"
- "外在威胁是不是主角内在恐惧的具象化？"

### 7.5 结局类型

| 类型 | 特征 | 网文适配 |
|------|------|----------|
| 完全解决 | 所有线索收束 | 独立卷/完结卷 |
| 开放式 | 留下问题给读者 | BE或风格化作品 |
| 苦甜 | 赢了但付出了代价 | 最有主题深度的选择 |
| 循环 | 回到起点但一切不同 | 适合重生流 |
| 钩子桥接 | 解决当前弧线+开启下一条 | **网文最常见的卷末结构** |

## 八、副线架构

### 副线的四种功能

1. **镜像副线：** 从不同角度反映主题。主线问"权力会腐蚀吗"，副线展示一个小角色被小权力腐蚀。
2. **对比副线：** 展示相反可能性。主角选择牺牲，副线角色选择自私并承担后果。
3. **复杂化副线：** 为主线增加压力和障碍。主角打反派的同时，后方出现政治危机。（术语统一于 subplot-management.md）
4. **起源副线：** 揭示背景故事，解释当前冲突的根源。

### 副线管理规则

- 长篇（50万字+）3-5条副线，短篇1-2条
- 每条副线必须连接主题，否则砍掉或合并
- 每条副线必须在高潮之前或高潮时解决（或明确留待续作）
- **检验：** 删掉这条副线，主线故事会缺什么？如果答案是"不会缺什么"，这条副线需要调整。

## 九、网文特有考虑

### 爽点设计

爽点是网文的结构性必需品，不是可选的锦上添花。

**爽点类型：**

| 类型 | 定义 | 示例 |
|------|------|------|
| 打脸型 | 当众证明质疑者是错的 | 被嘲笑废物的主角一招制敌 |
| 升级型 | 境界/能力突破 | 突破练气入筑基，天地异象 |
| 装逼型 | 意外展露隐藏实力 | 随手一击震惊所有人 |
| 反转型 | 等待已久的复仇/因果报应 | 曾经欺辱主角的人跪地求饶 |
| 认可型 | 得到重要人物的认可 | 宗主亲自收徒 |
| 守护型 | 成功保护了重要的人/物 | 以伤换伤保住师妹 |
| 智谋型 | 用智慧碾压对手 | 提前三步布局，对手自投罗网 |

**密度规则：** 至少每3-5章一个大爽点。小爽点可以填充间隔。大爽点前必须有足够的压抑积累。

**爽点内在化：** 纯外在爽点（打赢了）爽过就忘。有内在意义的爽点（打赢了而且是因为他克服了某个内心恐惧才打赢的）读者会反复回味。

### 钩子密度

每章结尾至少一个钩子。每5章一个大钩子。

| 钩子类型 | 示例 |
|----------|------|
| 悬念钩 | "谁杀了他父亲？"——未回答的问题 |
| 紧张钩 | "明天就是决战"——预期中的事件 |
| 发现钩 | "他不知道的是……"——戏剧性讽刺 |
| 矛盾钩 | "但所有人的反应都不对劲"——某处不对 |
| 悬崖钩 | 章节结束在动作/揭露的中间 |

### 黄金前三章

**第1章 — 钩住：** 用动作或悬念开局，不用设定说明开局。通过行为展示主角核心特质。结尾给一个翻到第2章的理由。

**第2章 — 落地：** 只扩展到够理解赌注为止。展示主角与众不同之处。给一个小爽点。

**第3章 — 承诺：** 引发事件或它的第一波涟漪到达主角。主角做出定义他是谁的选择。读者此时应理解：这是什么故事、这是谁、为什么要在意。结尾设置第一个大弧线的钩子。

## 十、常见情节问题与讨论方法

### 中段塌陷

**症状：** 引发事件和高潮之间失去动力。

**讨论方法：**
- "中段的主角是主动出击还是被动应对？如果太长时间都在被动，读者在等而不是在看。"
- "中段加一个'虚假胜利'或'意外挫折'——让读者以为要结束了然后又来一下。"
- "能不能设一个中期目标？看起来和最终目标一致但其实方向不同。"

### 机械降神

**症状：** 问题被外力、巧合或他人解决，不是主角的行动/选择。

**讨论方法：**
- "这个问题的解决方式——是主角自己解决的，还是'恰好'有人帮他？"
- "金手指不能是万能钥匙。金手指创造机会，但主角必须自己走过去。"

**规则：** 任何解决方案必须至少3章前有铺垫。主角必须为此付出代价。

### 被动主角

**症状：** 事情发生在主角身上。他们反应但不驱动。

**讨论方法：**
- "'被追杀所以逃跑'不算选择。'明知道危险但决定去'才算。"
- "给主角一个他自己设定的目标，哪怕这个目标是错的——错误的选择比没有选择有意思得多。"

## 十一、情节架构自检清单

在大纲完成前，确保能回答以下所有问题：

### 基础结构
- [ ] **主题：** 这个故事到底在说什么？
- [ ] **中央问题：** 用模板填写的具体问题，能通过检验标准？
- [ ] **大结构：** 选了哪种幕结构？为什么？
- [ ] **网文流派结构：** 用了哪种主线结构（升级/系统/重生/打脸）？节奏是否符合该流派的节拍要求？

### 冲突与转折
- [ ] **外部冲突：** 主角必须做什么？
- [ ] **内部冲突：** 主角必须成为什么？
- [ ] **双层冲突连接：** 外部冲突是内部冲突的具象化吗？
- [ ] **引发事件：** 什么时候？第3章之前？
- [ ] **中点转折：** 什么类型的转折？是否改变了故事方向？
- [ ] **高潮：** 双层冲突如何汇聚？主角的牺牲是什么？

### PASS/FAIL检验
- [ ] **每个转折 = 选择：** 用选择设计模板检验了所有关键转折点？
- [ ] **代价可见：** 每次前进都有失去？
- [ ] **高潮 = 牺牲：** 最终胜利的代价是什么？
- [ ] **无机械降神：** 所有关键问题的解决方案都有铺垫？

### 副线与节奏
- [ ] **副线：** 每条副线的功能是什么？删掉它主线会缺什么？
- [ ] **爽点分布：** 每3-5章至少一个大爽点？大爽点前有足够的压抑？
- [ ] **钩子密度：** 每章结尾有钩子？每5章有大钩子？
- [ ] **前三章：** 钩住、落地、承诺，三步都做到了？
