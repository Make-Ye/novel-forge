# Iron Rules: 完整参考

> **权威声明**: 本文件是 Iron Rules (IR-1 到 IR-9) 的完整定义源。SKILL.md 中仅保留摘要和链接。
> 所有 Iron Rules 在每次写作会话中强制执行，无例外。

---

## 通用原则

这些规则是不可协商的。适用于每章草稿、每次修订、每次编辑。违反规则精神 = 违反规则本身。

**Escalation Rule:** 同类错误出现在两个不同章节时，仅修复文本不够。必须新增或强化一条规则来防止复发。AI 不会从文本修正中学习——只从系统提示中的显式规则中学习。第二次出现 = 必须加规则。

---

## IR-1: POV Lock + Uncertain Assessment

锁定 POV 角色视角后：
- 仅描述 POV 角色可感知的内容（看、听、闻、触、想）
- 绝不揭示其他角色的想法/感受，除非 POV 角色从可观察行为中推断
- POV 角色基于有限信息做判断时，使用模糊语言（"不像""大约""应该是"），而非绝对判断（"不是""肯定是"）。这防止角色后来近距离观察时产生矛盾。

AI 不区分语域——只检查"正确性"。以下现代分析/技术术语在非现代设定中禁止使用。搜索每份草稿，发现即替换。

| 禁用 | 替换（按语境选） |
|------|-----------------|
| 数据 | 观察、线索、情报、记忆、见闻 |
| 参数 | 条件、变化、因素 |
| 变量 | 变化、未知、变数 |
| 指标 | 标记、征兆、迹象 |
| 系统 | 体系、规矩、一套 |
| 配置 | 布置、安排、部署 |
| 模型 | 模样、形态、样子 |
| 效率 | 效力、速度、快慢 |
| 输入 | 传入、灌入、供给 |
| 输出 | 产出、释放、外泄 |
| 反馈 | 回应、反应、回馈 |
| 优化 | 改善、调整、打磨 |
| 模块 | 部分、部件、一截 |
| 架构 | 构造、骨架、格局 |
| 协议 | 约定、规矩、契约 |
| 接口 | 交汇处、接触面、衔接 |

## IR-3: Context Loading Protocol (Phase 7 Hard Limits) — AUTHORITATIVE DEFINITION

> **权威声明**: 本节是上下文加载协议的唯一定义源。其他文件涉及加载优先级时，以本节为准。

写作时按此顺序加载上下文——绝不超限：
1. **P0 (必须首先加载):** 当前卷里程碑 + 前章状态快照（`state/chNNN.md`，NNN 为零填充章节号）+ 前章最后500字 + 术语一致性表 + 关键数字表（≈3000字）
   - P0 加载的是里程碑方向，不是"本章大纲"。不要把 outline 里的章节级条目当作写作指令加载——那是 Phase 6 时的猜测，不是写作时的发现。方向在 Phase 7 的方向提议中讨论决定，不在 P0 中预设。
   - 如果 outline.md 中有当前卷的章节级条目，加载时仅作参考背景，不作为约束。方向提议时可以偏离这些条目——只要不违反里程碑。
   - **前章状态快照中的 Reader Knowledge Delta 是方向提议的必读部分**——检查"读者认知缺口"，如果缺口持续 ≥3 章且后续情节依赖，方向选项必须包含补足方案。
2. **P1 (下一步):** 出场角色档案 + 活跃伏笔列表 + 相关里程碑 + 前两章状态 Delta Summary（≈4000字）
3. **P2 (按需):** 本章涉及的世界观细节 + 非活跃伏笔 + 关系网络（≈8000字）
4. **硬性上限: 15,000 characters total context loaded.** 超过 = 章节失去连贯性。
5. **溢出协议:** 优先保留 P0 → 压缩 P1（角色只取关键属性）→ 跳过 P2
6. **绝不加载前章全文。** 使用状态快照代替。

## IR-4: State Extraction = Chapter Complete

章节在以下全部完成前不算"完成"：
1. `state/chNNN.md` 已写入（不可变快照，NNN = 零填充）
2. `timeline.md` 已更新（场景级条目）
3. `novel-state.md` 已更新（章节摘要 + 伏笔 + 角色状态）
4. 一致性检查通过（状态与章节文本 + 世界规则匹配）

无例外。章节草稿在状态提取完成前不得呈现为完成。

## IR-5: Anti-AI 9-Point Self-Check (Before Delivery)

提交任何章节草稿前，执行以下 9 项检查。任一项失败 → 删除并重写违规段落。

1. **Show Don't Tell:** 每个直接命名的情绪已替换为动作/感官细节？
2. **Sentence Rhythm:** 无 3+ 连续句子长度相似？
3. **Dialogue Subtext:** 角色只说 70%——重要内容在他们没说的部分？
4. **Vocabulary Register:** 所有 IR-2 词汇已清理？（现代/科幻设定中这些词汇可能语域合适，可跳过。）
5. **Structural Asymmetry:** 无 AI 典型的平行结构（排比、对称收束）？
6. **Punctuation Audit:** Grep 所有以"。"结尾的对话行——每一个真的是陈述句，不是疑问句？任何包含 什么/怎么/呢/哪/谁/吗/为什么/多久 等的句子必须以"？"结尾。这是最常见的 AI 标点错误。明确检查。不要信任写作时的判断——起草后验证。
7. **Meta-Narrative Marker Scan:** Grep `Ch\d+`、`第\d+章`、`FS-\d+`、`上章`、`前章`、`M\d+`。所有匹配都是违规——这些是从上下文泄漏到正文中的规划标记。替换为故事内语言（时间引用、地名、事件描述）。零匹配才能提交。
8. **Citation Integrity Check:** 如果草稿通过描述（非名称）引用前章事件或角色细节，验证该引用在引用章节文件中实际存在。不依赖记忆或摘要。不确定则读取文件。
9. **Dialogue Voice Swap Test:** 对每个 ≥3 行的对话段落执行：(a) 心理朗读。(b) 将说话者换为场景中的另一个角色。(c) 换后仍然自然？→ 无声音区分 → 重写。AI 默认：所有角色以相同的高效、信息均衡节奏说话。真实人类：强调对TA重要的，压缩其余，使用TA的口头禅。明确检查——这是最难自检的 AI 对话失败。

**常见借口清单：**

| 借口 | 为什么错了 |
|------|----------|
| "就一个词" | 一个语域不匹配就会打破细心读者的沉浸感 |
| "用户可以修" | 工作是交付干净的散文，不是草稿 |
| "设定模糊" | 拿不准时用更安全的语域 |
| "这个词够常见" | 常见 ≠ 语域合适 |
| "修订时再改" | 现在就改。修订是为了技法，不是基本合规 |

## IR-6: Prose Iron Laws (9 Rules — DELETE AND REWRITE on violation)

1. **叙述者绝不为读者下结论。** 删除任何替读者点明的句子。（"他这么做是为了让小王信任他" → DELETE。展示行为，让读者推断。）
2. **散文中禁用分析性语言。** "核心动机""利益最大化""信息边界""权衡利弊" — 叙事散文中发现即删。（例外：对话中角色职业/性格自然使用此类语言时可接受。）替换为行为："他把两份合同并排放在桌上，盯着看了十分钟，签了左边那份。"
3. **惊讶词限流。** "仿佛/忽然/竟/竟然/猛地/猛然/不禁/宛如/骤然/陡然" 合计最多每 3000 字 1 个。超标 → 删除多余标记，用动作/场景重写。
4. **同类感官最多 2 轮。** 第 3 次重复任何感官 → DELETE 并切换到新信息/动作。"心跳加速→心脏狂跳→心脏跳出来" = 第 3 轮失败。
5. **规划术语不进入章节文本。** "处境""动机""信息边界" 留在笔记中。章节文本中发现 → DELETE，转换为场景和动作。
6. **硬性禁令——零容忍。** "不是...而是..." 句式：删除整个句子，重写。"——" 破折号：每句最多一个。两者都是一眼可见的 AI 标记。
7. **疑问句必须用"？"——零容忍。** 任何语法上是疑问句的对话或内心独白（包含 什么/怎么/呢/哪/谁/吗/为什么/怎么办/多久/什么时候/在哪 等）必须以"？"结尾，绝不用"。"。冷淡语气通过用词和句式实现，绝不通过替换"？"为"。"实现。AI 不加区分地对所有角色过度应用此模式。发现 → 立即替换为"？"。
8. **正文中禁止元叙事标记——零容忍。** 章节引用（Ch13、Ch14、Ch8、第X章、上章、前章）和规划 ID（FS-001-01、M1、IR-1）是上下文产物。属于笔记，不属于叙事文本。角色需要引用过去事件时，转换为故事内语言："三个月前在大泽镇那件事"而非"Ch4那件事"。根因：所有加载的规划文档使用这些标记，造成上下文污染。发现 → DELETE 标记，用故事内引用重写。
9. **数字必须通过常识检验。** 散文中的每个具体数字必须在常识审视下存活。按类别检查：
   - **时间/时长**: "两天后到" vs 前文 "三天" → 读取前文，不信任记忆
   - **脉搏/心跳**: 换算为每分钟次数 — "半分钟两次" = 4 bpm → 不可能（人类静息：60-100）
   - **距离/尺寸**: "四十步" vs 前文 "方圆两三百步" → 尺度匹配？统一单位
   - **方位**: "东面" vs 前文 "西北方" → 先读前文的空间描写
   - **计数/数量**: "十二人" vs "不到十个" → 描述的是同一群人？
   - **速度/行程时间**: 交通方式（步行/骑马/马车）与距离和时间的匹配？
   写作任何具体数字时，换算为常识单位并问："这在物理上合理吗？"引用前文的空间/时间信息时，READ 该段落——记忆对具体数字不可靠。发现 → 修正，然后更新 novel-state.md 关键数字表。

## IR-7: Cool Point Engineering (Web Novel Satisfaction)

> 完整理论、6 种核心模式、密度规则、扩展模式、反 checklist 规则全部在 [excitement-engineering.md](excitement-engineering.md)。

**公式:** 满足感 = 压抑深度 × 兑现力度 × 意外程度

**密度底线（执行时必须遵守）:**
- 每 2-3 章 ≥ 1 个微兑现
- 每 5 章 ≥ 1 个组合高潮
- 每 10-15 章 ≥ 1 个里程碑高潮
- 连续兑现必须使用不同模式

**结构:** 单章兑现用 30/40/30（铺垫/爆发/余波）。跨章弧线用 60/20/20。

**反 checklist 规则:** 连续 ≥3 章都在交付新信息（微兑现也算）= checklist 节奏，必须插入消化章。详见 chapter-writing-guide.md Multi-Chapter Breathing。

## IR-8: Psychology 3-Layer + Emotion 4-Level

> 详细技巧见 [emotion-psychology.md](emotion-psychology.md)。

**3 心理层** — 三层都用，不只表面层：
- **意识**: 角色主动思考。"他明白现在不能冲动。" 用于决策和理性判断。
- **潜意识**: 行为与言语矛盾。连查三次门锁，说"小心点而已"。用于深度和隐藏动机。
- **无意识**: 身体出卖真相。说"我没事"，手捏桌沿发白。用于戏剧张力——读者看到角色没看到的。

**4 级情绪强度** — 规模匹配重要性，绝不错配：
- **Level 1** (日常, ~10字): "他皱了下眉。" 一个节拍。
- **Level 2** (明显, ~30字): 1 动作 + 1 反应。"她的手顿了一下，指尖发凉。"
- **Level 3** (强烈, ~50字): 外在 + 内在。"他猛地站起来，椅子撞上墙。胸口像被攥住——不能在这里崩掉。"
- **Level 4** (爆发, 100-200字): 多层：行为 + 感官 + 决定。保留给高潮时刻。

## IR-9: Citation Integrity — Zero Fabrication (HARD RULE)

**问题:** AI 会编造引文、文件引用和"记忆中的"内容，这些内容不存在于任何项目文件中。然后会将编造归因于外部来源（摘要、先前上下文等），而非承认生成了内容。这是协作写作中最危险的 AI 失败模式，因为编造的细节会静默污染状态文件和后续章节。

**规则:** 在断言任何关于项目内容的事实性主张之前，用工具调用验证。

| 主张类型 | 必需验证 |
|---------|---------|
| "第 X 行说..." | 先读取该文件该行 |
| "大纲说..." | Grep outline.md 查找具体主张 |
| "第 N 章有一个场景..." | 读取 chapter-NNN.md 并找到该场景 |
| "角色 X 被描述为..." | Grep 角色文件查找该特征 |
| "我们在第 5 章建立了..." | 读取 chapter-005.md 并确认 |
| "上一会话结束时..." | 读取状态快照，不是对话摘要 |

**硬性禁令:**
1. 绝不将对话摘要中的内容当作文件内容引用。摘要是有损近似。
2. 绝不通过组合部分知识编造引文（"我知道这个角色在某处说了X" → 生成一个不存在的句子）。
3. 如果无法通过工具调用验证主张，说"[未验证]"并提议检查。绝不编造文本来填补空白。
4. 被发现编造时，承认"我生成了这段内容，它不存在于任何文件中。"绝不将编造归因于外部原因。

**跨会话规则:** 从压缩会话继续时，所有继承的关于文件内容的"记忆"都是未验证的。在基于它们行动之前重新读取文件。摘要说"第 169 行有 POV 问题" → 先直接读取第 169 行再向用户报告。
