# 副线生命周期管理（Subplot Lifecycle Management）

> **权威声明**: 本文件是子剧情管理的权威参考。
> 情节架构见 plot-architecture.md。三线追踪见 excitement-engineering.md。

> 副线是长篇叙事的骨架关节——没有它们，主线只是一根直线；管理不善，又会变成一团乱麻。
> 本文档提供副线从创建到收束的完整调度框架，确保 3-5 条副线与主线协调运转。
> 底层张力曲线模型见 pacing-curve-reference.md，本文件侧重副线如何在整体节奏框架中占位与调度。

---

## 一、副线定义与分类

### 1.1 主线 vs 副线

| 维度 | 主线（Main Plot） | 副线（Subplot） |
|---|---|---|
| 定义 | 核心冲突与目标，驱动全书叙事 | 次要冲突或情感线，丰富叙事层次 |
| 删除影响 | 故事不复存在 | 故事仍然完整，但厚度和感染力下降 |
| 篇幅占比 | 通常 50-70% | 单条占 5-15%，合计 30-50% |
| 视角 | 通常绑定主角 | 可绑定配角、反派、群像 |
| 收束要求 | 必须在结尾明确解决 | 可部分开放（续集伏笔） |

**判断标准**：如果删除这条线后故事的核心因果关系不受影响，它就是副线而非主线。

### 1.2 四种副线功能类型

每条副线必须承担至少一种明确功能：

| 类型 | 功能 | 设计要点 | 适用场景 |
|------|------|----------|----------|
| **镜像副线** | 反映主线主题的反面/侧面，从不同角度理解同一主题 | 与主线主题同构但走向相反，形成互文 | 需要深化主题时优先使用 |
| **对比副线** | 与主线形成张力，制造叙事落差和节奏变化 | 基调、节奏、情感方向与主线形成显著反差 | 需要调节节奏或增加呼吸感 |
| **复杂化副线** | 给主线增加困难、障碍或道德困境 | 必须直接影响主线的因果链 | 需要提升紧迫感或引入新变量 |
| **起源副线** | 解释主线发生的原因——回溯动机、历史、因果 | 信息量必须对主线有补充价值 | 需要揭示深层背景时 |

> **规则**：每条副线创建时标注主功能类型。允许兼具两种功能，但主功能只能有一个。

### 1.3 副线层级

| 层级 | 持续范围 | 篇幅占比 | 推进频率 | 功能限制 |
|---|---|---|---|---|
| A级 | 贯穿全书 | 10-15% | 每 5 章推进一次 | 必须与主线有 ≥2 个交叉点 |
| B级 | 单卷（约 20-40 章） | 5-10% | 每 8 章推进一次 | 必须与本卷主线有 ≥1 个交叉点 |
| C级 | 短段（5-10 章） | 3-5% | 每 10 章推进一次 | 可独立存在，但需与主线有关联 |

**层级提升**：C级表现优异可升B级，B级可升A级。A级不可降级——失去动力的A级副线应修剪。

### 1.4 副线数量指南

| 作品规模 | 章节范围 | A级上限 | B级上限 | C级上限 | 总计上限 |
|---|---|---|---|---|---|
| 短篇 | < 30 章 | 1 | 1 | 0 | ≤ 2 |
| 中篇 | 30-100 章 | 2 | 2 | 1 | ≤ 4（推荐 3） |
| 长篇 | 100+ 章 | 3 | 3 | 2 | ≤ 6（推荐 4-5） |

> **原则**：宁可少而精，不要多而散。每增加一条副线，管理复杂度呈指数增长。

## 二、副线贡献评分系统

### 2.1 为什么需要评分

"感觉这条副线没什么用"是模糊的判断。贡献评分用具体标准量化每条副线的价值，让修剪/保留/升级决策有据可依。

### 2.2 五维评分标准

每条副线在每个维度上打 0-5 分，权重不同，总分 0-100：

| 维度 | 权重 | 5分标准 | 0分标准 |
|------|------|---------|---------|
| **主线推进力** | 30% | 直接推动主线核心冲突发展 | 与主线因果链完全无关 |
| **角色深化度** | 25% | 揭示关键角色的深层特质或产生重大变化 | 不涉及任何角色的内在变化 |
| **冲突烈度** | 20% | 制造高压困境或道德两难 | 没有实质性冲突或张力 |
| **读者回报率** | 15% | 提供情感高潮或关键信息揭示 | 读者可以跳过而不影响理解 |
| **主题共振** | 10% | 与全书核心主题形成互文或深化 | 与核心主题无关联 |

### 2.3 评分操作指南

```
每个维度 0-5 分打分后：
  加权总分 = 主线推进力×6 + 角色深化度×5 + 冲突烈度×4 + 读者回报率×3 + 主题共振×2
  满分 = 30+25+20+15+10 = 100

决策阈值：
  ≥ 70分 → GREEN：健康副线，持续投入
  40-69分 → YELLOW：需加强，找到薄弱维度并改善
  20-39分 → ORANGE：启动修剪评估（见第五节）
  < 20分 → RED：立即修剪或合并

注意：评分应在副线进入生长期（GROW）后进行。
种子期和发芽期的副线分数天然偏低，不应过早修剪。
```

### 2.4 各功能类型的预期强项

| 功能类型 | 应该高分（≥4）的维度 | 可以低分（≤2）的维度 |
|----------|----------------------|----------------------|
| 镜像副线 | 主题共振、角色深化度 | 主线推进力 |
| 对比副线 | 读者回报率（节奏调节） | 冲突烈度 |
| 复杂化副线 | 主线推进力、冲突烈度 | 主题共振 |
| 起源副线 | 读者回报率（信息揭示） | 冲突烈度 |

> 如果副线在其"应该高分"的维度得分 ≤2，说明功能类型标注错误或设计失败。

### 2.5 评分实践示例

```
示例副线："陈探员追查妹妹失踪案"（复杂化副线，B级）
  主线推进力：4 × 6 = 24（发现妹妹失踪与连环案有关，推进主线调查）
  角色深化度：5 × 5 = 25（揭示陈探员创伤、动机、道德困境）
  冲突烈度：4 × 4 = 16（在查案和保护妹妹之间产生两难）
  读者回报率：3 × 3 = 9（中等悬念，读者想知道妹妹下落）
  主题共振：2 × 2 = 4（与全书主题关联一般）
  总分：78 → GREEN，健康副线
```

## 三、副线生命周期与转换条件

### 3.1 五阶段总览

```
种子期(SEED) → 发芽期(SPROUT) → 生长期(GROW) → 高潮期(CLIMAX) → 收束期(RESOLVE)
  ↑                                                                |
  └────────────── (修剪/重置，仅限CLIMAX之前) ←────────────────────┘
```

### 3.2 各阶段定义与持续时间

| 阶段 | 定义 | A级持续 | B级持续 | C级持续 |
|------|------|---------|---------|---------|
| SEED | 埋设伏笔，读者不察觉是独立副线 | 3-8章 | 2-5章 | 1-3章 |
| SPROUT | 副线获得独立驱动力，读者能察觉 | 5-10章 | 3-7章 | 2-4章 |
| GROW | 副线与主线交叉，产生影响 | 10-20章 | 5-12章 | 3-6章 |
| CLIMAX | 副线自身冲突爆发 | 2-4章 | 2-4章 | 2-4章 |
| RESOLVE | 副线与主线汇合或独立解决 | 3-6章 | 2-4章 | 1-2章 |

### 3.3 转换条件（必须全部满足才能推进阶段）

#### SEED → SPROUT（种子发芽）

必须满足：
- [ ] 已埋设 ≥2 个伏笔锚点
- [ ] 种子元素已在 ≥3 个不同场景中自然出现
- [ ] 副线已获得至少一个专属角色或独立场景
- [ ] 读者此时能感知到"这里有条新的故事线"（检查：如果读者仍完全没察觉，继续播种）

#### SPROUT → GROW（发芽进入生长）

必须满足：
- [ ] 副线拥有独立的驱动力（不再只是主线的附属品）
- [ ] 至少一个角色因此副线发生了可察觉的变化或产生了新的目标
- [ ] 读者已产生"想知道这条线怎么发展"的好奇心
- [ ] 能明确指出一个即将与主线产生的交叉点

#### GROW → CLIMAX（生长进入高潮）

必须满足：
- [ ] 至少完成了一个与主线的交叉点（因果关系已建立）
- [ ] 副线冲突已积累到无法回退的临界点
- [ ] 关键角色已面临必须做出选择的局面
- [ ] 没有其他副线正在经历高潮期（高潮隔离规则）
- [ ] 距离全书主线高潮还有足够空间（至少5章以上完成收束）

#### CLIMAX → RESOLVE（高潮进入收束）

必须满足：
- [ ] 核心冲突已爆发，关键选择已做出
- [ ] 收束方向已明确（汇合主线 / 独立解决 / 悬而未决）
- [ ] 收束方式与副线的功能类型一致

#### RESOLVE → 完成

必须满足：
- [ ] 给了读者一个满意的交代（即使是开放式收束也必须有阶段性满足感）
- [ ] 副线引入的悬置元素已全部清理
- [ ] 追踪板状态更新为 RESOLVED

> **回退规则**：只有 SEED→SPROUT 和 SPROUT→GROW 可以回退。CLIMAX 之后不可逆——如果进入高潮后发现方向错误，必须走完收束期，不能"假装没发生过"。

## 四、副线调度系统

### 4.1 副线追踪板格式

```markdown
| 字段 | 值 |
|---|---|
| 副线名称 | [填写名称] |
| 功能类型 | 镜像/对比/复杂化/起源 |
| 层级 | A/B/C |
| 当前阶段 | SEED/SPROUT/GROW/CLIMAX/RESOLVE |
| 起始章节 | 第 X 章 |
| 当前章节 | 第 Y 章 |
| 上次推进 | 第 Z 章 |
| 下次计划 | 第 W 章（计划推进内容简述） |
| 与主线交叉点 | [章节列表和交叉描述] |
| 预计收束章节 | 第 N 章 |
| 贡献评分 | [最近一次评分/日期] |
| 状态 | ACTIVE / PAUSED / PRUNED / RESOLVED |
```

### 4.2 推进频率规则

| 层级 | 标准间隔 | 最大容忍间隔 | WARN | CRITICAL |
|---|---|---|---|---|
| A级 | 每 5 章 | 8 章 | 8 章 | 12 章 |
| B级 | 每 8 章 | 12 章 | 12 章 | 16 章 |
| C级 | 每 10 章 | 15 章 | 15 章 | 20 章 |

**触发动作**：
- `WARN`：下次大纲规划时必须安排推进
- `CRITICAL`：立即推进或启动修剪评估

### 4.3 并发推进限制

1. **单章推进上限**：同一章最多推进 2 条副线
2. **连续推进限制**：同一批副线不能连续 3 章同时推进
3. **高潮隔离**：同一时间只能有 1 条副线处于高潮期

### 4.4 副线与主线的交织模式

| 模式 | 图示 | 适用 | 风险 |
|------|------|------|------|
| 交替推进 | 主线→副线→主线→副线 | 需要明显节奏变化 | 切换过于频繁打断沉浸感 |
| 嵌入式推进 | 主线场景中嵌入副线碎片 | 种子期/发芽期，不宜占独立章节 | 嵌入过浅读者感知不到 |
| 汇流式推进 | 多条副线与主线汇合 | 多副线同时收束 | 执行难度高，容易显得刻意 |

> 汇流式推进最多同时涉及 3 条线（含主线），超过必须错开。

### 4.5 调度决策流程

每规划一个新章节时按序检查：

1. 主线当前状态——是否需要密集推进？
2. 各副线距上次推进的章节数——是否触发警告？
3. 副线阶段——是否有线处于或即将进入高潮期？
4. 并发限制——当前章是否已经推了 2 条副线？
5. 交织模式——当前适合交替、嵌入还是汇流？

## 五、副线修剪指南

### 5.1 修剪触发条件

使用贡献评分系统（第二节）+ 以下信号综合判断：

| 信号 | 判定标准 | 优先级 |
|---|---|---|
| 贡献评分 RED | 加权总分 < 20 | HIGH |
| 贡献评分 ORANGE 且无改善 | 连续2次评分 < 40 且未改善 | HIGH |
| 长期停滞 | 超过 15 章无推进（任何层级） | HIGH |
| 主线脱节 | 无法指出与主线的交叉点或关联 | HIGH |
| 篇幅膨胀 | 占比超出层级上限的 150% | HIGH |
| 功能重叠 | 与另一条副线承担相同功能且评分更低 | MEDIUM |
| 读者冷感 | beta 读者反馈中无人提及此副线 | MEDIUM |

### 5.2 修剪方法选择决策树

```
这条副线需要修剪。它目前处于什么阶段？
│
├─ SEED 或 SPROUT 阶段
│   ├─ 功能与其他副线重叠？ → 合并到其他副线（Merge）
│   ├─ 读者已经注意到它？ → 快速收束（1-2章内给出交代）
│   └─ 读者几乎没注意到？ → 降级为背景元素（Background）
│
├─ GROW 阶段
│   ├─ 有明确的收束路径？ → 快速收束（2-3章内合理结束）
│   ├─ 核心元素可转移？ → 合并到其他副线
│   └─ 已建立的世界观有价值？ → 降级为背景元素 + 保留世界观贡献
│
└─ CLIMAX 或 RESOLVE 阶段
    └─ 禁止修剪。必须走完收束期。
```

### 5.3 修剪方法详解

| 方法 | 用时 | 适用 | 注意事项 |
|------|------|------|----------|
| 快速收束 | 1-2章 | C级、发芽期/生长期早期 | 不是草率收束——必须回应已建立的读者预期 |
| 合并 | 立即 | 功能重叠的副线 | 合并后重新评估层级和贡献评分 |
| 降级为背景 | 立即 | 起源副线（已完成背景信息任务） | 相关角色不应再占用显著篇幅 |

### 5.4 修剪禁忌

1. 不能无理由消失——已建立的副线必须给读者某种交代
2. 不能在高潮期修剪——已进入 CLIMAX 必须走完收束
3. 不能一次修剪超过一条——同时砍多条会让叙事结构崩塌
4. 修剪后必须更新追踪板——标记 `PRUNED` 并记录方式和章节

## 六、副线高潮对齐

### 6.1 核心原则

**副线高潮不应与主线高潮同时发生。** 同时爆发的多条高潮会互相稀释情感投入。

### 6.2 理想排序

```
副线高潮 A → [间隔 1-2 章] → 副线高潮 B → [间隔 1-2 章] → 主线高潮
```

### 6.3 错开规则

| 场景 | 最小间隔 | 推荐间隔 |
|---|---|---|
| 两条副线高潮之间 | 3 章 | 4-5 章 |
| 副线高潮与主线高潮之间 | 1 章 | 2-3 章 |
| 同一副线内次要高潮之间 | 5 章 | 8-10 章 |

### 6.4 收束方式选择

| 收束方式 | 适用 | 要点 |
|----------|------|------|
| 与主线汇合 | 复杂化/起源副线 | 汇合前给读者足够线索 |
| 独立解决 | 镜像/对比副线 | 确保不显得突兀 |
| 悬而未决 | A级跨书副线 | 必须给"阶段性"满足感，每书最多1-2条 |
| 悲剧性收束 | 复杂化副线 | 必须让读者感到"有意义"而非"白费了" |

## 七、副线与三线交织的关系

### 7.1 概念区分

| 维度 | 三线（Quest/Fire/Constellation） | 副线 |
|---|---|---|
| 本质 | 叙事基调维度（行动/情感/世界观） | 独立故事线 |
| 作用 | 决定"怎么讲" | 决定"讲什么" |

### 7.2 副线的三线配比

每条副线可以有不同的三线配比：

| 副线类型 | Quest（行动线） | Fire（情感线） | Constellation（世界观线） |
|----------|----------------|----------------|--------------------------|
| 主线 | 70% | 15% | 15% |
| 感情副线 | 10% | 80% | 10% |
| 世界观副线 | 20% | 20% | 60% |
| 复杂化副线 | 60% | 30% | 10% |

### 7.3 配比设计要点

- 不同副线的三线配比应有明显差异（>20%差异），否则副线之间会感觉雷同
- 镜像副线：Fire 占比较高；对比副线：与主线形成反差；复杂化副线：Quest 占比较高；起源副线：Constellation 占比较高
- 配比可随阶段自然调整（如感情副线在高潮期 Quest 比例上升）
- 两条副线配比相似度 >80% 时触发警告，应考虑合并或调整

## 八、FAIL/PASS 示例

### FAIL：仅为了填充篇幅而存在的副线

```
副线："主角的同事小王每天中午吃什么的日常"
  主线推进力：0 × 6 = 0（与主线调查完全无关）
  角色深化度：1 × 5 = 5（小王是功能性配角，无内在变化）
  冲突烈度：0 × 4 = 0（今天吃面还是吃饭不是冲突）
  读者回报率：0 × 3 = 0（读者可以全部跳过）
  主题共振：0 × 2 = 0（与"正义与代价"的核心主题无关）
  总分：5 → RED
诊断：这条副线的唯一目的是"让书更长"。它没有功能类型，
      无法标注为镜像/对比/复杂化/起源中的任何一种。
      删除后主线完全不受影响。
```

### PASS：改变主角理解的副线

```
副线："退休老刑警张叔的个人救赎——他当年办过一桩冤案"
（镜像副线，A级）
  主线推进力：3 × 6 = 18（张叔的冤案线索帮助主角重新审视当前案件）
  角色深化度：5 × 5 = 25（张叔从自保到主动坦白，完成角色弧线；
                同时让主角看到"坚持正义的代价"，改变对案件的态度）
  冲突烈度：4 × 4 = 16（张叔在隐瞒过去和弥补过错之间痛苦挣扎）
  读者回报率：4 × 3 = 12（冤案真相揭示是全书最感人的场景之一）
  主题共振：5 × 2 = 10（直接呼应全书核心主题"正义的代价"）
  总分：81 → GREEN
诊断：这条副线从反面（冤案=正义的失败）映照主线（查案=追求正义），
      读者通过张叔的故事重新理解主角的挣扎。
      删除后不仅主线失去一个关键信息来源，主题深度也会大幅下降。
```

### FAIL：多条副线同时进入高潮

```
第25章安排：
  主线高潮：主角与反派正面对决
  副线A高潮：感情线破裂
  副线B高潮：搭档背叛真相揭露
后果：读者在3000字内要同时消化三个情感峰值，
      每一个都来不及感受就被下一个冲走了。
      三条线的情感互相稀释，没有一个能留下深刻印象。
诊断：违反"高潮隔离"规则，同时违反"单时间只1条高潮"规则。
```

### PASS：高潮错开排列

```
第22章：副线B（搭档背叛线）高潮 + 收束开始
第23章：副线B收束完成，主线推进
第24章：副线A（感情线）高潮
第25章：副线A收束 + 主线加速推进
第26-27章：主线高潮
后果：每个情感峰值都有独立的情感空间。
      读者先消化搭档背叛的冲击，再感受感情破裂的痛楚，
      最后以充沛的情感投入迎接主线最终对决。
      副线的收束为读者提供了情感缓冲，增强了主线高潮的力度。
```

### FAIL：从未与主线交叉的副线

```
副线C："主角的邻居养了一只流浪猫的成长故事"
  从第3章引入，写到第30章，猫从幼猫长成大猫。
  主线推进力：0（猫的故事从未影响主线调查）
  与主线交叉点：0
  存在15章后的状态：CRITICAL（远超C级最大容忍间隔）
诊断：这条副线与主线完全平行，永远不会交叉。
      读者会困惑"为什么要告诉我猫的事"。
      应在第10章左右触发修剪评估并快速收束。
```

### PASS：每条副线都有明确的交叉设计

```
副线C："主角收养的流浪猫经常跑到隔壁废弃工厂"
（看似日常，实为C级复杂化副线）
  种子期（第3-5章）：猫的行为引发主角注意到废弃工厂
  发芽期（第6-8章）：主角去工厂找猫，发现可疑痕迹
  生长期（第9-12章）：痕迹指向反派的秘密仓库——与主线交叉
  高潮期（第13章）：猫在仓库遇险，主角被迫提前暴露
  收束期（第14章）：猫安全，但主角发现了关键证据
  贡献评分：60+ → YELLOW 偏 GREEN（短小但有效）
诊断：表面是"养猫日常"，实际每一步都在推进主线。
      猫不是填充篇幅的宠物，而是叙事工具——它让主角
      以"找猫"的自然理由去到主线需要他去的地方。
```

## 附录：副线设计自检清单

创建新副线前，回答以下问题：

1. **必要性**：删除这条线后，主线是否会变得单薄？如果是，它值得存在
2. **功能**：它属于哪种功能类型？功能是否明确？
3. **层级**：它应该是什么层级？持续多长时间？
4. **交叉**：它将在哪个章节与主线交叉？交叉方式是什么？
5. **配比**：它的三线配比是什么？是否与现有副线有足够差异？
6. **转换条件**：你能否预见它从 SEED → SPROUT → GROW → CLIMAX → RESOLVE 的路径？每个阶段的转换条件大致能达成吗？
7. **收束**：它将如何收束？收束方式是否与功能类型一致？
8. **资源**：你是否有足够的篇幅推进它？是否会挤占主线或其他副线的空间？
9. **贡献预期**：根据评分系统预估，它能否达到 GREEN（≥70分）？

如果问题 1-6 无法明确回答，或问题 9 预估 < 40分，暂缓创建这条副线。
