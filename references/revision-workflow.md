# 修订工作流指南

> novel-forge 技能的修订流程参考手册。定义从单章微调到跨书级联变更的完整修订协议。

---

## 核心原则

| 原则 | 说明 |
|------|------|
| **修旧章不改旧章** | 用状态快照追踪影响范围，而非重读全文后凭印象修改 |
| **最小侵入式修订** | 采用 str_replace 模式精准替换，禁止全文重写（除非必要） |
| **级联变更向前传播** | 任何修改的影响必须沿章节链向前传播，直到无冲突为止 |
| **同步三件套** | 正文、state/chNNN.md、timeline.md 三者必须保持一致 |
| **修订留痕** | 每次修订在 state 文件中追加修订日志，确保可追溯 |

---

## 一、修订触发类型

| 触发来源 | 典型场景 | 默认响应级别 |
|----------|---------|-------------|
| **用户主动修改** | "把第 5 章那个场景改一下" | 按用户指定范围 |
| **审查报告** | 编辑流程发现一致性问题 | 按问题严重程度 |
| **设定回溯** | 世界观/角色核心设定变更 | 全局级联扫描 |
| **情节漏洞** | 连载后读者反馈或作者自查 | 评估影响范围后决定 |
| **风格统一** | 声音编辑发现叙述语气不一致 | 中修订起步 |

**响应级别定义**：

- **小修订**：局部措辞/描写调整，不改变情节走向
- **中修订**：场景片段重写，涉及角色行为或情绪变化
- **大修订**：情节线变更、角色命运调整、世界观规则修改

## 二、单章修订流程

### 2.1 诊断阶段

```
1. 读取目标章节正文
2. 读取对应 state/chNNN.md
3. 必要时读取前后各一章的 state 文件（判断衔接影响）
4. 明确修订范围
```

### 2.2 范围判定

| 判定条件 | 修订模式 | 操作方式 |
|----------|---------|---------|
| 仅修改措辞/描写细节 | 局部修改 | str_replace 精准替换 |
| 修改对话/角色反应，但不改变情节走向 | 局部修改 | str_replace + 小范围补充 |
| 场景需要重排或扩写 | 片段重写 | 替换目标段落块 |
| 情节走向改变或角色命运变更 | 全章重写 | 按 Phase 7 流程重写（跳过方向提案） |

### 2.3 局部修改检查清单

- [ ] 替换后的文本与上下文衔接自然
- [ ] 修改未引入与 state 文件矛盾的新信息
- [ ] 角色称呼、语气与角色卡一致
- [ ] 时间线逻辑未被破坏
- [ ] 伏笔/暗示未被意外移除

### 2.4 全章重写检查清单

- [ ] 核心情节节点与大纲保持一致
- [ ] 保留未被修改的伏笔和暗示
- [ ] 章首衔接前章章尾
- [ ] 章尾衔接后章章首
- [ ] 所有角色出场与 state 文件记录匹配

### 2.5 执行修订后

无论修订大小，完成后**必须**执行 Phase 8（状态重新提取）：

```
1. 重新提取 state/chNNN.md
2. 与旧版 state 对比，确认变更已被记录
3. 检查 timeline.md 对应条目是否需要更新
```

## 三、级联修订协议

当修改第 N 章影响后续章节时，必须执行级联修订。

### 3.1 级联触发条件

| 修改类型 | 是否触发级联 |
|----------|-------------|
| 纯文字润色 | 否 |
| 角色新增伤势/外貌变化 | 是（后续场景需体现） |
| 角色关系状态改变 | 是（后续互动需调整） |
| 地点描述变更 | 可能（取决于后续是否出现） |
| 情节结果改变 | 是（严重） |
| 世界规则/设定变更 | 是（全局） |
| 伏笔内容变更 | 是（后续回收需对应） |

### 3.2 级联修订步骤

```
步骤 1：提取变更 Delta
    - 从旧 state 和新 state 的差异中提取变更清单
    - 分类：角色状态 / 关系 / 物品 / 地点 / 时间线 / 设定

步骤 2：扫描受影响章节
    - 读取第 N+1 到最新章的所有 state 文件
    - 逐项检查每个 state 文件是否与变更冲突
    - 生成受影响章节列表

步骤 3：标记冲突
    - 对每个受影响章节，列出具体冲突点和修复建议
    - 按从近到远排序（先修 N+1，再修 N+2……）

步骤 4：逐章修复
    - 修复章节正文（str_replace 模式）
    - 更新对应 state 文件
    - 更新 timeline.md 条目

步骤 5：验证链完整性
    - 确认所有 state 文件无残留矛盾
    - 必要时运行 Continuity Editor 跨章检查
```

### 3.3 级联修订影响评估表

| 影响维度 | 检查内容 | 优先级 |
|----------|---------|--------|
| 角色状态 | 伤势、位置、持有物是否延续正确 | 高 |
| 关系网络 | 互动态度是否符合变更后的关系 | 高 |
| 情节因果 | 后续事件的触发条件是否仍然成立 | 高 |
| 时间线 | 事件顺序和时间间隔是否仍然合理 | 中 |
| 环境描写 | 场景细节是否与修改后的描述一致 | 中 |
| 伏笔回收 | 修改是否影响已埋设或已回收的伏笔 | 高 |

## 四、回溯修改同步规则

修改已完成章节时，**必须同步更新**以下三个文件：

### 4.1 同步三件套

| 文件 | 更新内容 | 更新时机 |
|------|---------|---------|
| **章节正文** | 修改内容本身 | 立即 |
| **state/chNNN.md** | 状态快照（角色、关系、物品、时间线等） | 正文修改完成后立即 |
| **timeline.md** | 对应时间线条目 | 如果修改涉及时间/事件变化 |

### 4.2 同步检查清单

- [ ] 正文已修改
- [ ] state/chNNN.md 已重新提取并覆盖旧版
- [ ] timeline.md 对应条目已更新（如适用）
- [ ] 修订日志已追加到 state 文件末尾
- [ ] 已评估是否需要级联修订（第三章）
- [ ] 如需级联，已生成受影响章节列表

### 4.3 禁止事项

- **禁止**只改正文不改 state 文件
- **禁止**直接修改旧版 state 而不保留变更记录
- **禁止**跳过级联评估直接结束修订
- **禁止**在不确定影响范围时假设"应该没问题"

## 五、设定变更扩散

当世界观或角色核心设定发生根本性变更时（如：修改魔法体系规则、改变角色核心性格、调整历史背景），需要执行设定扩散流程。

### 5.1 扩散步骤

```
步骤 1：源头更新
    - 在 world.md 或 characters/ 对应文件中更新设定
    - 在设定旁标注变更原因和变更日期

步骤 2：全章扫描
    - 搜索所有已写章节中涉及该设定的引用
    - 使用关键词/实体名称搜索
    - 标记每一处引用的章节和上下文

步骤 3：冲突分类
    - 对每处引用判定：一致 / 轻微矛盾 / 严重矛盾
    - 按矛盾严重程度排序

步骤 4：按优先级修复
    优先级排序：
    a. 直接矛盾的情节关键点
    b. 角色行为与设定冲突
    c. 世界规则违反
    d. 环境描写细节
    e. 提及/暗示级别的引用

步骤 5：批量更新 state
    - 更新所有受影响章节的 state 文件
    - 追加修订日志
```

### 5.2 设定变更影响范围评估

| 设定类型 | 典型影响范围 | 扫描策略 |
|----------|-------------|---------|
| 魔法/能力体系 | 所有涉及该能力的章节 | 关键词 + 能力名称 |
| 角色核心性格 | 该角色出场的所有章节 | 角色名 + 行为描写 |
| 地理/建筑布局 | 涉及该地点的所有章节 | 地点名称 + 场景描写 |
| 历史事件/背景 | 涉及该背景的所有引用 | 事件关键词 |
| 物品/道具属性 | 涉及该物品的所有章节 | 物品名称 |
| 势力/组织关系 | 涉及该势力的所有章节 | 势力名 + 成员名 |

## 六、修订与编辑系统的集成

不同规模的修订应触发不同级别的编辑审查。

### 6.1 编辑触发规则

| 修订规模 | 必须运行的编辑 | 可选编辑 |
|----------|---------------|---------|
| **小修订** | 一致性编辑 | — |
| **中修订** | 一致性 + 节奏 + 声音编辑 | 读者模拟编辑 |
| **大修订** | 完整六大编辑流程 | — |
| **设定变更后** | 一致性编辑（跨章检查） | 按需追加其他编辑 |
| **级联修订后** | 一致性编辑（所有受影响章节） | 按章节改动量追加 |

### 6.2 六大编辑角色速查

> 权威定义见 `editorial-roles.md`，以下为快速参考。

| 编辑角色 | 核心关注点 | 详细参考 |
|----------|-----------|---------|
| 一致性编辑（Continuity） | 与前文/设定保持一致 | `editorial-roles.md` Section 1 |
| 节奏编辑（Pacing） | 场景比例与张力曲线 | `editorial-roles.md` Section 2 |
| 声音编辑（Voice） | 角色语言独特性 | `editorial-roles.md` Section 3 |
| 故事编辑（Story） | 情节逻辑与叙事有效性 | `editorial-roles.md` Section 4 |
| 读者模拟编辑（Reader Sim） | 读者体验与期待管理 | `editorial-roles.md` Section 5 |
| 重复检测编辑（Repetition） | 词汇/结构/模式重复 | `editorial-roles.md` Section 6 |

### 6.3 编辑运行注意事项

- 级联修订中，**先完成所有正文修复**，再统一运行编辑检查
- 编辑发现问题后的修复本身也需要更新 state 文件
- 设定变更扩散完成后，Continuity Editor 应对全部已写章节做一次扫描

## 七、版本追踪

### 7.1 修订日志格式

每次修订完成后，在对应章节的 state 文件末尾追加修订日志：

```markdown
## 修订日志

| 日期 | 修订类型 | 修改内容摘要 | 影响章节 |
|------|---------|-------------|---------|
| 2026-05-22 | 局部修改 | 修正第3章主角左臂伤势描述（灼伤→擦伤） | Ch3, Ch4 |
| 2026-05-22 | 设定变更 | 调整灵力体系：取消"灵力上限"概念 | Ch1-Ch8 |
```

### 7.2 字段说明

| 字段 | 填写要求 |
|------|---------|
| **日期** | 修订执行的日期（YYYY-MM-DD） |
| **修订类型** | 局部修改 / 片段重写 / 全章重写 / 设定变更 / 级联修复 |
| **修改内容摘要** | 一句话概括修改了什么，以及为什么修改 |
| **影响章节** | 列出所有被修改的章节编号（含级联影响） |

### 7.3 版本追踪原则

- 每次修订**必须**追加日志，不允许删除或覆盖旧日志
- 级联修复的每一章都要在各自的 state 文件中追加日志
- 日志按时间倒序排列（最新的在最前面）
- 当日志超过 20 条时，考虑将旧日志归档到单独文件

## 附录：修订工作流快速决策树

```
收到修订请求
  │
  ├─ 仅涉及单章？
  │    ├─ 是 → 判定修改范围（局部/片段/全章）
  │    │        → 执行修改 → 更新 state → 评估级联 → 结束
  │    │
  │    └─ 否（涉及设定变更）→ 执行设定扩散流程（第五章）
  │
  ├─ 是否改变情节走向？
  │    ├─ 是 → 大修订 → 全章重写 → 级联评估 → 完整编辑流程
  │    └─ 否 → 评估影响范围 → 按规模执行 → 对应级别编辑
  │
  └─ 是否影响后续章节？
       ├─ 是 → 提取 Delta → 扫描 state 文件 → 级联修复
       └─ 否 → 单章修订 → 更新三件套 → 结束
```
