# Novel-Forge 写前研究 Briefing 系统

> **权威声明**: 本文件是写前简报和量化质量标准的权威参考。
> 上下文加载协议以 SKILL.md IR-3 为准。

## 核心理念

**写前不做研究，等于蒙着眼睛开车。** 每章动笔之前，必须先完成一份结构化的写前Briefing，确保AI掌握了所有必要的上下文，不会凭空编造。

## 零、Planner 15条工作原则（从InkOS移植）

写前Briefing的底层逻辑。每条原则指导Briefing的生成方向。

| # | 原则 | 在Briefing中的体现 |
|---|------|------------------|
| 1 | **读者第一** — 每个决定先问"读者会怎样" | Briefing的段2（读者期待）优先级最高 |
| 2 | **承诺必兑现** — 埋了钩子就必须回收 | Briefing的段3明确列出"该兑现的" |
| 3 | **节奏即生命** — 张弛有度比持续高潮更重要 | Briefing确认本章张力目标（张力理论见 pacing-curve-reference.md，大纲级节奏见 outline-template-guide.md，本文件仅覆盖章前检查） |
| 4 | **少即是多** — 留白比填满更有力 | Briefing标注"不写的部分" |
| 5 | **矛盾驱动一切** — 没有矛盾就没有故事 | Briefing确认本章的核心矛盾 |
| 6 | **角色大于剧情** — 剧情为角色服务不是反过来 | Briefing确认角色行为符合其性格 |
| 7 | **感官锚定真实** — 具体感官细节建立信任 | Briefing指定本章的主导感官 |
| 8 | **信息差即张力** — 控制信息释放制造好奇 | Briefing标注"暂不掀的"信息 |
| 9 | **每章一个改变** — 章尾和章头必须有区别 | Briefing段6强制回答"什么变了" |
| 10 | **开头三行定生死** — 读者决定是否继续就在前三行 | Briefing选择开篇技术 |
| 11 | **结尾钩子保命** — 章尾决定读者是否翻下一章 | Briefing选择收束技术 |
| 12 | **对话是潜台词** — 说出来的不是最重要的 | Briefing标注本章对话的潜台词方向 |
| 13 | **反套路本能** — 每次想用套路时先问"有更好的吗" | Briefing检查本章是否包含经典套路 |
| 14 | **三线不偏废** — 主线/感情/世界观交替推进 | Briefing确认本章的线条分配 |
| 15 | **写完回头看** — 写完后用编辑视角重检 | 章节后检查清单执行 |

## 零点五、黄金三章法则

前三章是全书最关键的章节。读者在前三章内决定是否继续阅读。

### 第一章：钩住

```
必须做到：
- 第一段就让读者产生"然后呢？"的好奇
- 前500字建立主角的核心处境（不需要完整背景）
- 章尾必须有至少一个Strong级别的钩子
- 字数控制在2000-3000字（短于常规章节）

禁止：
- 大段背景/世界观介绍
- 超过3个新角色同时出场
- 平淡的日常开场（"清晨，阳光洒在小村庄..."）

开篇技术推荐（任选一种）：
A. 动作开局：直接从一个有张力的场景开始
B. 悬念开局：一个反常现象或一句话
C. 对话开局：一句引人注意的话
D. 感官开局：一个强烈的感官细节
```

### 第二章：确认

```
必须做到：
- 兑现第一章的部分承诺（不是全部）
- 展示主角的"特殊之处"（不一定是能力，可以是性格/处境/视角）
- 至少一个让读者"意外"的元素
- 字数可以比第一章稍长

禁止：
- 用第二章做世界观大段介绍
- 让主角被动超过全章50%
- 节奏和第一章完全相同
```

### 第三章：绑定

```
必须做到：
- 给读者一个"这就是这书的风格"的确认
- 完成第一个小弧线（有铺垫→爆发→余波的完整循环）
- 至少一个微兑现或小爽点
- 明确核心矛盾/冲突的方向

禁止：
- 还不展示本书的核心卖点
- 三个配角以上同时新出场
- 和前两章的基调完全断裂
```

### 黄金三章强度指标

| 指标 | Ch1 | Ch2 | Ch3 |
|------|-----|-----|-----|
| 张力起点 | 4+ | 3+ | 5+ |
| 张力峰值 | 7+ | 6+ | 8+ |
| 信息密度 | >1.5点/千字 | >1.2点/千字 | >1.5点/千字 |
| 钩子强度 | Strong | Medium+ | Strong |
| 角色出场 | ≤3人 | ≤2新人 | ≤2新人 |

## 一、写前Briefing七段格式（从InkOS Planner移植）

每章写作前生成一份7段式memo，作为写作指令：

### 段0：弧线位置感知（写作前先定位）

> 在写任何内容之前，先确定本章在当前2-3章弧线单元中的位置。这决定了本章是"给信息"还是"消化信息"。

```
最近三章信息交付回顾：
- Ch.[N-2]: [交付了什么新信息/事件]
- Ch.[N-1]: [交付了什么新信息/事件]
- Ch.[N]:   本章属于哪个位置？

弧线位置（三选一）：
□ 蓄力章 — 本弧线的起点。埋种子、建张力、深化角色、建立赌注。不急于兑现。
□ 转折/兑现章 — 本弧线的核心。兑现前章积累的张力，让某件事发生。不要犹豫。
□ 消化章 — 本弧线的呼吸。角色反应、展示后果、消化前章的冲击。不引入新的重大信息。

诊断：如果最近2+章都是"蓄力章"（都在给新信息），本章必须是"消化章"。
```

**核心规则：连续≥3章都在交付新信息 = checklist 节奏 = 必须插入消化章。** 读者需要时间感受后果，才能对下一个信息产生情绪投入。详见 chapter-writing-guide.md Multi-Chapter Breathing。

### 段1：当前任务（一句话）

> 一句话概括本章要完成什么。

示例："林墨在拍卖会上意外发现师父遗留的玉简，但竞争对手是赵家嫡子。"

### 段2：读者此刻在等什么

> 上一章结尾给读者的承诺/期待是什么？读者带着什么心情翻开本章？

```
需要回答：
- 上一章的钩子是什么？
- 读者最想知道什么？
- 如果本章不给，读者会不会失望？
```

### 段3：该兑现的 / 暂不掀的

```
该兑现的（本章必须处理）：
- [前章埋下的具体期待]
- [已超过5章未推进的伏笔]
- [读者等待中的信息]

暂不掀的（本章不能揭露）：
- [计划在后续章节揭示的重大信息]
- [需要更多铺垫才能引爆的伏笔]
- [如果现在揭露会浪费后续张力的事件]
```

### 段4：日常/过渡承担什么任务

如果本章包含过渡或日常场景（低张力），这些场景不能只是"过路"，必须承担至少一项功能：

| 功能 | 说明 |
|------|------|
| 角色深化 | 通过日常互动揭示角色新面向 |
| 世界观扩展 | 自然地展示新的设定细节 |
| 关系推进 | 小规模的关系变化或暗示 |
| 伏笔埋设 | 在轻松场景中埋下重要伏笔 |
| 对比蓄力 | 用平静反衬即将到来的风暴 |

**禁忌：** 纯过渡场景不得超过本章篇幅的20%。如果超过，要么砍掉要么给它一个任务。

### 段5：关键抉择过三连问

如果本章包含角色的关键决策，必须回答：

```
1. 这个抉择为什么难？（必须在两个好东西或两个坏东西之间选）
2. 每个选项的代价是什么？（没有代价的选择不叫选择）
3. 选择了之后，不可逆的后果是什么？（选择必须有重量）
```

### 段6：章尾必须发生的改变

> 本章结束时，什么是和本章开始时不同的？

```
至少一项：
- 角色状态变化（新能力/新信息/新关系/新处境）
- 世界状态变化（新势力/新规则/新地理）
- 读者认知变化（读者知道了之前不知道的事）
- 情感状态变化（关系升温/信任崩塌/希望燃起）
```

**禁忌：** 如果章尾和章头没有明显变化，这章可能不需要存在。

### 段7：本章Hook账

> 钩子ID 格式为 `FS-[章节]-[序号]`（如 FS-003-01）。操作与伏笔四态的映射：Open → PLANTED / Advance → WATERED / Resolve → HARVESTED / Defer → 不改变状态。详见 consistency-system.md Section 9.4。

```
Open（本章新埋的钩子）：
- [FS-XXX-NN] - 预计回收：[弹性范围，如"Vol.2中段""10-15章内""下个高潮弧线"]（⚠ 不用固定章节号，避免制造隐形合同）
- [FS-XXX-NN] - 预计回收：[弹性范围]

Advance（本章推进的已有钩子）：
- [FS-XXX-NN] - 推进方式：[如何推进]

Resolve（本章兑现的钩子）：
- [FS-XXX-NN] - 兑现方式：[如何兑现]

Defer（本章故意压住不动的钩子）：
- [FS-XXX-NN] - 压住理由：[为什么现在不掀]
```

## 二、写前上下文加载协议（从webnovel-writer context-agent移植）

### 2.1 五段式任务书

在生成7段memo之前，先通过研究模式收集上下文：

**第一段：开篇委托**
- 本章在全书大纲中的位置
- 本章前后的章标题和一句话梗概（前后各3章）
- 本章在大纲中的原始目标

**第二段：故事上下文**
- 前一章 state 文件（`state/chNNN.md`）—— 角色位置、状态、关系变化、伏笔更新
- 前章最后500字原文（衔接语感）
- 前两章 state 文件的 Delta Summary（快速回顾）
- 当前活跃的伏笔列表（从 novel-state.md 读取）
- 未解决的角色困境

**第三段：人物**
- 本章出场角色的档案（从 `characters/` 对应角色文件读取）
- 这些角色当前的知识状态（从前章 state 文件读取）
- 角色之间的关系温度（从前章 state 文件读取）
- 角色最近的重大经历（从前章 state 文件的 Changes This Chapter 读取）
- **角色 Verbal Signatures**（从角色卡 Voice Profile 读取）：本章有对话的每个角色至少确认两个语言特征维度（句长偏好/口头禅/信息优先级/正式度），写作时保持区分（IR-5 Check 9）

**第四段：怎么写更顺**
- 本章的张力级别（来自大纲）
- 推荐的开篇技术
- 推荐的收束技术
- 需要避免的重复（与最近3章的开头/结尾对比）
- **场景新鲜度检查**：回顾最近3-5章的主要地点和场景类型，本章是否需要切换地点或场景类型？连续≥3章同一主要地点或同一场面类型（如连续对话、连续战斗）= 必须在本章切换（见 chapter-writing-guide.md Cross-Chapter Scene Freshness）
- **数字/方位验证清单**：本章是否涉及时间线、距离、方位、脉搏、速度等具体数字？如果是，列出需要与前文交叉验证的项目，并标注来源章节（IR-6 Rule 9）

**第五段：收在哪里**
- 本章必须到达的终点事件
- 章尾建议的钩子类型
- 下一章的开头预设（确保衔接）

### 2.2 上下文容量控制

**总量限制：15000字以内**（指加载的上下文，不含本章正文）

优先级分配（权威定义见 SKILL.md IR-3，此处为简化摘要）：
- P0（必须加载）：本章大纲 + 前章状态快照 + 前章最后500字 + 术语表 + 关键数字表 ≈ 3000字
- P1（应该加载）：出场角色档案 + 活跃伏笔 + 前两章 Delta Summary ≈ 4000字
- P2（按需加载）：世界观细节 + 非活跃伏笔 + 关系网络 ≈ 8000字

**关键规则：绝不加载前章全文。** 改用状态快照 + 最后500字衔接。如超出15000字，优先保留P0 → 压缩P1 → 跳过P2。详见 SKILL.md IR-3。

如果超出15000字：
1. 优先保留P0
2. 压缩P1（角色档案只取关键属性，state 文件只取 Delta Summary）
3. 跳过P2

## 三、量化质量标准（综合各工具最佳实践）

### 3.1 章节质量10大标准（从AI-Practical-Lab/novel-writer移植）

每章完成后，按以下10项评分（每项2分，总分20分）：

| # | 标准 | 2分(优秀) | 1分(及格) | 0分(不及格) |
|---|------|----------|----------|------------|
| 1 | 开场抓人 | 前3句建立张力或好奇 | 前10句进入正题 | 开头拖沓无趣 |
| 2 | 节奏紧凑 | 无明显拖沓段 | 拖沓段<15% | 大段无推进 |
| 3 | 冲突有力 | 冲突升级，有转折 | 有冲突但平 | 无实质冲突 |
| 4 | 角色动机 | 行为有深层理由 | 行为有表面理由 | 行为无理由 |
| 5 | 对话鲜活 | 有潜台词+声音区分 | 有功能但平淡 | 说明式对话 |
| 6 | 结尾钩子 | 读者必须翻下一章 | 有一定吸引力 | 可读可不读 |
| 7 | 信息密度 | >1剧情点/千字 | 1剧情点/千字 | <0.7剧情点/千字 |
| 8 | 画面感 | 有"只有此场景才有"的细节 | 有具体描写 | 通用的、可替换的描写 |
| 9 | 情感传递 | 通过行为/感官传递情感 | 有情感但直述 | 无情感或标签化 |
| 10 | 去AI味 | 通过所有Anti-AI指标 | 大部分通过 | 明显AI痕迹 |

**判定标准：**
- 80%+ (16分以上)：优秀，可进入下一章
- 60%+ (12分以上)：良好，小修后进入下一章
- 40%+ (8分以上)：需要大幅修改
- <40% (8分以下)：建议重写

### 3.2 Anti-AI通过标准

章节必须同时满足：

| # | 检查项 | 阈值 |
|---|--------|------|
| 1 | 高风险词汇密度 | <= 5个/千字 |
| 2 | 段落长度变异系数CV | >= 0.15 |
| 3 | 转折词段落内重复 | < 3次 |
| 4 | 连续同开头结构 | < 3句 |
| 5 | 自然化指标达标项 | >= 4/6项 |

### 3.3 自然化指标标准

| 指标 | 不达标 | 达标 | 优秀 |
|------|--------|------|------|
| 停顿词频率 | <0.5次/500字 | 1-2次/500字 | 2-3次/500字 |
| 不确定表达 | 0次/章 | >=2次/章 | 3-4次/章 |
| 短句占比(<=10字) | <20% | 30-50% | 40-60% |
| 口语词频率 | 0次/1000字 | >=2次/1000字 | >=4次/1000字 |
| 碎片句(2-4字独立) | <1次/章 | 3-5次/章 | 5-8次/章 |
| 对话中断/被打断 | 0次/章 | 1-2次/章 | 2-4次/章 |

## 四、预写检查清单（综合增强版）

写每章前的8项必检：

```
[ ] 1. 写前Briefing已生成（七段memo）
[ ] 2. 上下文已按协议加载（P0+P1完成）
[ ] 3. POV角色已锁定（知识状态已审查）
[ ] 4. 活跃伏笔已审查（该兑现的列出来了）
[ ] 5. 前章状态快照已加载（角色位置/状态确认）
[ ] 6. 场景分解已完成（2-3个场景，比例分配好）
[ ] 7. 开篇和收束技术已选择（与最近3章不重复）
[ ] 8. 张力目标已确认（与大纲一致）

If any item is missing, STOP and resolve it before writing.
```

### 章节后检查清单

```
[ ] 1. 章节正文已保存
[ ] 2. 10大质量标准已自评（>=12分）
[ ] 3. Anti-AI 5项检查已通过
[ ] 4. 自然化指标已检查（>=4项达标）
[ ] 5. 状态提取已完成（10步协议）
[ ] 6. 伏笔台账已更新
[ ] 7. 四表已更新（道具/时间/术语/数字）
[ ] 8. Hook账已更新（open/advance/resolve/defer）
```
