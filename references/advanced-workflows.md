# 高级工作流系统

> **权威声明**: 本文件是高级工作流（写作教练、爆剧情模式、解构代理、场景指令、银魂法角色设计、Bootstrap 导入）的权威参考。
> 提案系统见 proposal-system.md。Anti-AI 防御见 anti-ai-system.md。

## 核心理念

**AI不仅是生成器，还可以是教练、爆炸手、解构师。** 本文件整合5种高级工作流模式，扩展novel-forge的创作能力。

## 一、写作教练模式（从opencode-novel-plugin移植）

### 1.1 章节开头选择

每章开始前，AI提供2-3个不同的开头版本：

```
版本A（动作开局）：
直接从张力场景切入，短句为主，建立紧迫感。
[生成的开头段落]

版本B（感官开局）：
用一个强烈的感官细节建立氛围，让读者"闻到""听到"场景。
[生成的开头段落]

版本C（对话开局）：
从一句引人注意的对话开始，制造好奇心。
[生成的开头段落]

标注：每个版本使用了什么开头技术，各有什么效果。
```

用户选择后AI继续写，或融合多个版本。

### 1.2 技巧标注

写作过程中，在关键段落旁标注使用的技法：

```
[技法：潜台词注入] — 这里通过"你家不路过这儿"暗示主角知道对方在关心自己，
而不是直接说"我知道你在关心我"。

[技法：感官桥过渡] — 用茶杯的热度从对话场景过渡到回忆场景。

[技法：不完美结尾] — 章尾不总结，停在一个未完成的状态。
```

**标注频率**：不是每段都标，仅在关键转折、高光、难点段落标注（每章3-5处）。

### 1.3 关键时刻双版本

在关键场景（告白、战斗、真相揭露、重大抉择）提供两个版本：

```
版本A（内心独白驱动）：
重点放在角色内心活动，用自由间接引语展示心理变化。

版本B（对话+动作驱动）：
重点放在外部行为和对话，通过潜台词暗示内心。

分析：
A适合：情感深度、角色理解、慢节奏
B适合：张力、冲突、快节奏
```

### 1.4 写后自我评估

完成后AI给出具体评论（不是打分）：

```
本章优点：
1. [具体段落引用] — 这个转折处理得自然，读者不会觉得突兀
2. [具体段落引用] — 对话有潜台词，两人说的和想的不一样

最满意的段落：
[引用] — 因为...

可以改进的地方：
[引用] — 这里的情感展示偏直白，可以考虑用动作替代
```

## 二、Booming情节爆炸模式（从vibe-noveling移植）

用于情节陷入瓶颈或需要高强度转折时。

### 2.1 四维正交组合

| 维度 | 选项 |
|------|------|
| **锚点**（爆炸什么） | 策略层/认知层/承诺层/关系层/力量层/信息层/时间层/规则层 |
| **机制**（怎么爆炸） | 反转/连锁/转移/侵蚀/崩塌/碰撞/梁移除/越界/错位/蒸发 |
| **尺度**（多大） | 个人/关系/势力/区域/世界/超自然/元叙事 |
| **不可逆**（能回来吗） | 可逆但加倍代价/部分可逆/完全不可逆 |

### 2.2 生成规则

1. 生成10个方向
2. 任意两个不能共享相同的锚点+机制组合
3. 至少2个必须"真正掀桌子"（角色死亡、重大挫折、失去身份/地位/盟友/身体/尊严）
4. 验证测试：**"如果这件事发生了，故事的后续方向会和之前完全不同吗？"**
   - 如果只是"情绪更强烈但所有人做的事一样"→ 不合格
   - 必须改变至少一个角色的行动轨迹

### 2.3 示例

```
方向1：
锚点：关系层 | 机制：背叛 | 尺度：势力 | 不可逆：完全不可逆
"主角最信任的副手其实是敌方卧底，而且已经卧底了三年。
 主角在这三年里分享的所有秘密全部泄露。"
验证：主角从此不再信任任何人 → 后续所有关系互动都改变 ✓

方向2：
锚点：力量层 | 机制：崩塌 | 尺度：个人 | 不可逆：可逆但加倍代价
"主角的核心能力突然失效——不是因为被压制，
 是因为这能力的代价终于到期了。要用回来，必须付出双倍。"
验证：主角必须重新学习不用这能力战斗 → 战斗方式完全改变 ✓

方向3：
锚点：认知层 | 机制：反转 | 尺度：世界 | 不可逆：完全不可逆
"主角发现一直以来的敌人其实是在保护世界，
 而主角以为的盟友才是真正的威胁。整个'善恶'框架颠倒。"
验证：主角必须重新选择阵营 → 所有关系和目标重新定义 ✓
```

## 三、解构Agent（从webnovel-writer移植）

分析参考小说，提取可迁移的技法（不是照搬故事）。

### 3.1 快速模式

适用于快速参考学习：

```
输入：参考小说的前3章（或用户提供文本）

输出：
1. 黄金前3章分析
   - 开头技术：用了什么开局方法
   - 钩子类型：第一章结尾的钩子类型和强度
   - 信息释放节奏：前3章释放了多少信息，留了多少

2. 结构解构
   - 弧线形状：渐强/骤变/波浪
   - 转折点位置：第几章出现第一个转折
   - 节奏模式：快-慢-快的节奏分布

3. 可迁移模式（3-5个）
   - 每个模式：具体描述 → 什么条件下可以应用 → 注意事项
```

### 3.2 深度模式

适用于系统性学习：

```
输入：整部小说文本或详细大纲

输出：
1. 逐章推进点提取（每章10-15个推进点）
2. 角色弧线追踪
3. 情节线归并（相关性≥0.7自动归入同一弧线）
4. 质量门槛：置信度≥0.85、覆盖率85-95%、重叠≤35%

抽象层：
- 将具体情节事件转化为条件框架
  "主角在密室发现真相" → "主角在封闭环境中获得颠覆性信息"
- 识别核心套路的边界（什么范围内有效，超出就变成俗套）
- 提取"展示与对比"模式
```

### 3.3 防止原著污染

```
do_not_copy清单：
- 不复制具体角色名、地点名
- 不复制具体的对话或段落
- 不复制独特的故事组合（可以有类似元素，不能有相同组合）

canon_contamination_warnings：
- 如果提取的模式与目标作品有冲突，标注警告
- 如果提取的技法依赖于原著特有的设定，标注限制条件
```

## 四、9部分场景写作指令（从opencode-novel-plugin移植）

与pre-writing-brief互补——brief是章节级，这是场景级。每个场景写前生成。

| 部分 | 内容 | 格式要求 |
|------|------|---------|
| 1. 角色 | 出场角色的关键状态 | 每人1-2句，聚焦当前状态而非全貌 |
| 2. 任务 | 这个场景要完成什么 | 1句话，可验证的目标 |
| 3. 角色特征 | 需要体现的角色特质 | 每人1-2个关键词 |
| 4. 情感 | 场景的情感基调+转变方向 | 起点情感 → 终点情感 |
| 5. 感官 | 主导感官+辅助感官 | 2种，不重复上一场景的感官组合 |
| 6. 节奏 | 节奏类型 | 紧凑/舒展/渐强/骤变，选一种 |
| 7. 避免 | 本章必须避免的模式 | 3-5项，来自最近3章的重复检测 |
| 8. 约束 | 不可违反的设定规则 | 具体条目，从world.md和characters/提取 |
| 9. 成功标准 | 怎样算写好了 | 2-3个可验证条件 |

### 场景指令示例

```
场景：林墨在拍卖会上发现师父的玉简

1. 角色：
   林墨——刚突破炼气期，灵力不稳，心情忐忑
   赵家嫡子——嚣张，财大气粗，对林墨有敌意

2. 任务：林墨发现拍卖品中有师父的遗物，但赵家嫡子也盯上了

3. 角色特征：
   林墨——克制但内心翻涌、观察力强
   赵家嫡子——张扬、不把人放在眼里

4. 情感：紧张期待 → 震惊发现 → 焦虑无力 → 决心

5. 感官：视觉（拍卖品的细节）+ 触觉（攥紧的拳头）

6. 节奏：渐强——从平静的拍卖氛围逐步升级到林墨的内心风暴

7. 避免：
   - 不要用"他的心跳加速"
   - 不要让林墨直接说"那是师父的东西"
   - 不要超过3个新角色同时出场

8. 约束：
   - 师父的玉简在设定中是玄级中品
   - 林墨当前灵石不足以竞拍
   - 赵家在拍卖行有VIP席位

9. 成功标准：
   - 读者能感受到林墨的焦虑但不说出"焦虑"
   - 拍卖会的紧张感传递给读者
   - 场景结尾读者想知道林墨怎么拿到玉简
```

## 五、银魂法角色设计（从vibe-noveling移植）

通过5种"记忆点"创造令人难忘的角色。

### 5.1 五种记忆点

| 类型 | 定义 | 设计要求 | 示例 |
|------|------|---------|------|
| **视觉记忆点** | 独特的外在特征 | 1-2个，可重复出现，可被其他角色吐槽 | 总是穿一只红袜子、脖子上有道疤形状像闪电 |
| **行为记忆点** | 独特的动作习惯 | 只有这个角色会做的事 | 紧张时把硬币在指间翻滚、思考时用左手摸右耳 |
| **语言记忆点** | 独特的说话方式 | 口头禅3-5字最佳，标志性句式 | 每句话结尾加"嘛"、总用反问句、喜欢用比喻 |
| **反差记忆点** | 日常vs关键时刻的反差 | 平时一个样，关键时刻完全不同 | 平时懒散到极点→战斗时优雅精准；平时话痨→说真话时只用一个字 |
| **梗（Running Gag）** | 反复出现的搞笑模式 | 需要2+角色互动，可以进化 | 每次见面对方都说"你怎么又瘦了"、总是被某种东西砸到 |

### 5.2 行为底层逻辑

每个核心角色必须回答一个问题：

> **"这个角色在任何情况下都不会放弃什么？"**

这个"不可妥协的执念"决定角色的所有选择：
- 被逼到绝境时做什么 → 取决于执念
- 面对诱惑时怎么选 → 取决于执念
- 和别人冲突时退不退 → 取决于执念

### 5.3 角色设计示例

**角色：苏暮，流浪剑客**

```
视觉记忆点：
- 左手无名指上缠着一根红绳，从不取下
- 背上的剑鞘比剑短半尺（剑露出一截）

行为记忆点：
- 每到一个新地方，先找最高的地方站一会儿（看地形）
- 吃饭前一定把筷子在桌角齐一下（强迫症级别的整齐）

语言记忆点：
- 不说"不知道"，说"没想好"
- 形容什么都用吃的比喻："那家伙比隔夜的馒头还硬"

反差记忆点：
- 平时吊儿郎当、嘴贱、能坐着绝不站着
- 拔剑的瞬间整个人沉默下来，所有多余的动作消失

梗：
- 每到一个新城镇都会迷路，偏偏不承认，说"我在绕路观察地形"

不可妥协的执念：
- 绝不欠人情。可以被打败、受伤、丢脸，但不能欠别人的。
  如果欠了，必须找机会还，哪怕还的代价远超所欠的。
```

**角色：沈青禾，药铺掌柜**

```
视觉记忆点：
- 总是穿深蓝色对襟褂子，袖口磨得发白
- 眼镜架在头顶但从来不戴（其实不近视，戴着显得专业）

行为记忆点：
- 说话时手永远不闲着：要么擦柜台、要么理药瓶、要么盘一串旧珠子
- 给人把脉的时候会闭眼哼一首无名的调子

语言记忆点：
- 叫所有人都叫"小X"（不管对方多老）
- 说教的时候喜欢用"我师父说过..."开头

反差记忆点：
- 平时和蔼可亲、笑眯眯、好说话
- 一旦有人在他药铺里动手，笑容不变但眼神变了，
   之后那个人在镇上看什么病都没人敢接

梗：
- 每次有人问他年龄，他都说"比上次见到你的时候大一岁"

不可妥协的执念：
- 药不能乱卖。不管对方是谁、出多少钱、用什么威胁，
   不该用的药绝不出手。宁可药铺关门也不违背这条。
```

### 5.4 记忆点设计检查清单

```
[ ] 1. 每个核心角色有至少3种记忆点
[ ] 2. 视觉记忆点可以被其他角色吐槽或引用
[ ] 3. 行为记忆点在紧张场景中仍会出现（强化角色一致性）
[ ] 4. 语言记忆点不只是口头禅——包含句式和用词偏好
[ ] 5. 反差记忆点不是"性格转变"，而是同一性格的不同面向
[ ] 6. 梗至少涉及2个角色，不是自娱自乐
[ ] 7. 不可妥协的执念能解释角色过去的至少一个重大决定
[ ] 8. 两个角色的执念不会完全相同
```

## 六、Bootstrap 导入工作流

当用户已有部分素材（大纲、角色卡、世界观笔记等）时，通过本流程导入并跳到合适的 Phase。

### 6.1 导入评估三步

**第一步：素材盘点**

询问用户提供所有已有素材，按以下维度分类：

| 素材类型 | 对应 Phase | 对应文件 |
|----------|-----------|---------|
| 类型/基调/目标读者 | Phase 1-2 | project-config.md, title-synopsis.md |
| 世界观/设定笔记 | Phase 3 | world.md |
| 角色设定/人物卡 | Phase 4 | characters/ |
| 剧情大纲/结构 | Phase 5-6 | plot.md, outline.md |
| 已写章节 | Phase 7 | chapters/ |
| 状态记录 | Phase 8 | state/, novel-state.md |

**第二步：完整度判定**

| 已完成的 Phase | 跳入点 | 说明 |
|---------------|--------|------|
| 无（只有灵感） | Phase 1 | 正常流程 |
| 有类型/世界观概念 | Phase 2 | 先定核心概念和书名 |
| 有完整世界观 | Phase 4 | 跳过 Phase 3，审查世界观后进入角色设计 |
| 有角色但无世界观 | Phase 3 | 先完成世界观，再审查角色是否与世界观一致 |
| 有世界观 + 角色 | Phase 5 | 跳过 Phase 3-4，审查后进入剧情架构 |
| 有完整大纲 | Phase 7 | 跳过 Phase 3-6，审查大纲后直接写第一章 |
| 有已写章节 | Phase 7（续写） | 从最后完成章节之后继续 |

**第三步：质量审查**

对用户提供的每份素材执行：
1. **格式映射**：将素材内容映射到对应模板的字段中
2. **一致性检查**：验证素材内部是否自洽（角色设定与世界观是否矛盾、大纲与角色动机是否匹配）
3. **缺失识别**：标出模板中存在但素材中缺失的字段
4. **向用户确认**：列出映射结果、发现的矛盾、缺失字段，请用户确认或补充

### 6.2 导入操作流程

```
1. 创建项目目录
2. 将素材内容填入对应模板文件（不是直接复制粘贴，而是按模板格式整理）
3. 标记已跳过 Phase 的阶段进度为 "imported"（而非 "completed"）
4. 在 novel-state.md 会话历史中记录导入操作
5. 对当前 Phase 执行正常 PRE-LOAD 和工作流
6. 如果后续发现导入素材有问题，随时回退修正
```

### 6.3 导入注意事项

- **"imported" vs "completed"**：跳过的 Phase 标记为 "imported"，表示基于用户素材跳过而非经过完整对话流程。后续如需修改，应回到该 Phase 重新执行。
- **不信任未验证的素材**：即使是用户提供的内容，也要经过一致性检查。用户可能记错自己写过的设定。
- **保留原始素材**：导入前将用户原始素材存入项目目录的 `import/` 文件夹，以备回溯。
- **已写章节的特殊处理**：如果用户提供了已写章节，需要对每章执行 Phase 8（状态提取），确保 state/ 文件齐全。缺少状态快照的已写章节会导致续写时上下文缺失。
