# Outline Template Guide: Milestone-Based Outline with Volume-Level Pacing

## Core Principle

The outline is a ROADMAP, not a SCRIPT. It defines where the story must go (milestones) and trusts the writing process to discover how to get there. Per-chapter detail is filled during Phase 7 through collaborative discussion — never pre-scripted.

> **权威参考声明：** 张力曲线的理论模型、反单调规则、特殊节奏技巧等均在 `pacing-curve-reference.md` 中定义。本文档覆盖 APPLICATION 层——如何将张力理论转化为大纲中的里程碑结构。

---

## Why Milestones, Not Chapters

**The problem with per-chapter outlines:**
- AI reads detailed plans as execution orders → writes to fulfill specs, not to create
- Pre-planned tension numbers feel mechanical when translated to prose
- Detailed outlines become straightjackets — the story can't surprise the writer
- Foreshadowing planned 50 chapters ahead often doesn't survive contact with actual writing

**What milestones give you:**
- Direction without restriction: "this volume must end with the protagonist entering the city"
- Creative space: the HOW is discovered during Phase 7 discussion
- Flexibility: if the story wants to go somewhere unexpected between milestones, it can
- Organic foreshadowing: planted during writing, not pre-scheduled

## Milestone Format

Every volume in the outline must include:

```
## Volume [N]: [Volume Title]

### Volume Overview
| Field | Content |
|-------|---------|
| **Central Question** | 这一卷回答什么问题 |
| **Start State** | 这一卷开始时主角/核心角色的状态 |
| **End State** | 这一卷结束时必须达到的状态 |
| **Tension Direction** | 整体张力走向 (rising / peaking / falling / volatile) |
| **New Setting Elements** | 这一卷将引入的新设定（写作中补入world.md） |

### Milestones (3-5 per volume)
| # | Milestone | What MUST Happen | Narrative Purpose | Approx. Chapter |
|---|-----------|-----------------|-------------------|----------------|
| M1 | [名称] | [必须发生的事件] | [这个里程碑的叙事功能] | ~Ch.XX |
| M2 | [名称] | ... | ... | ~Ch.XX |
| ... | ... | ... | ... | ... |

### Open Threads Carried Forward
| Thread | From Volume | Status | Target Resolution |
|--------|------------|--------|-------------------|
| [ID] | [描述] | [活跃/延迟/已解] | [预计在哪一卷收回] |
```

### Field Definitions

**Central Question** — The one question this volume answers for the reader. NOT a plot summary.
- NOT: "主角在灰市站稳脚跟"
- YES: "一个没有力量的废人如何在灰色地带靠信息生存？"

**Tension Direction** — The overall shape, not per-chapter numbers:
- Rising (↗): each milestone raises the stakes
- Peaking (⇈): volume builds to a climactic moment
- Falling (↘): aftermath volume, processing consequences
- Volatile (↕): alternating highs and lows

**Milestone Narrative Purpose** — Why this event MATTERS:
- Revelation (揭露): changes what the reader/character understands
- Escalation (升级): raises the stakes
- Transformation (转变): changes a character's trajectory
- Confrontation (对抗): forces a decisive encounter
- Discovery (发现): opens a new path or information

## FAIL/PASS: Milestone Entry Examples

### Central Question

```
FAIL:
  Central Question: 主角去灰市
  问题：这是剧情描述，不是问题。读者不会因为"去灰市"而产生好奇。

PASS:
  Central Question: 一个什么都没有的废人，凭什么在灰市活过第一周？
  为什么好：有悬念、有压力、暗示了生存主题。
```

### Milestone

```
FAIL:
  M1: 主角到达灰市，找到钱家情报铺子，和钱鹤年谈判。
  问题：这是场景级细节，不是里程碑。预先规定了过程，没有留下发现空间。

PASS:
  M1: 灰市立足 — 主角在灰市找到第一个盟友（是谁、怎么找到的 = 写作中发现）
  为什么好：定义了结果（找到盟友），开放了过程。写作时的讨论决定具体怎么发生。
```

### Tension Direction

```
FAIL:
  Ch1:4  Ch2:5  Ch3:7  Ch4:9  Ch5:6 ...
  问题：逐章预设张力值。AI会把这些当指标去执行，写出来的节奏是算出来的。

PASS:
  Tension Direction: Rising (↗) — 从陌生环境的不安感起步，经过第一次危机建立威胁，
  在M3达到小高潮（身份差点暴露），M5是本卷最大危机（韩昭的追杀令传到灰市）。
  为什么好：描述了读者体验的走向，不是数字序列。具体章节的节奏在Phase 7讨论中决定。
```

## Chapter Entry Format

Every chapter in the outline must include ALL of the following fields:

```
### Chapter [Number]: [Title]

| Field | Content |
|-------|---------|
| **Purpose** | 这个章节在故事中的功能（一句话） |
| **Tension Level** | [1-10] （10为最高） |
| **POV** | 视角角色 |
| **Key Events** | 本章发生的核心事件（2-3条） |
| **Emotional Target** | 读到这里应该感受到什么 |
| **Continuity Hook** | 本章承接上文什么 |
| **Setup for Next** | 为下文铺垫了什么 |
| **Foreshadowing** | 本章埋下的伏笔/收回的伏笔 |
| **爽点标记** | 大爽/小爽/无（网文专用） |
| **Estimated Words** | 预估字数 |
```

### Field Definitions

**Purpose (章节功能)** — The structural reason this chapter exists:
- Setup (铺设) / Escalation (升级) / Revelation (揭露) / Action (行动)
- Emotional (情感) / Transition (过渡, use sparingly) / Climax (高潮)
- Resolution (收束) / Reflection (沉淀)

**Tension Level (紧张度 1-10)** — Reader's anxiety level:
- 1-3: Calm, establishing, reflective
- 4-5: Building, suspicious, slightly uneasy
- 6-7: Active threat, conflict, urgency
- 8-9: Peak danger, emotional crisis, major revelation
- 10: Climactic moment, maximum stakes

**Emotional Target (情感目标)** — What the reader should FEEL. Be specific:
- NOT: "excited" or "sad"
- YES: "为主角的处境揪心但又相信他能翻盘" or "终于看到反派露出破绽的痛快"

**Foreshadowing (伏笔)** — Track both planting and harvesting with unique IDs (格式: FS-[章节]-[序号]):
- Planting: "[FS-001-01] 在此处埋下：..."
- Harvesting: "[FS-001-01] 在此处收回：..."

## FAIL/PASS: Field Entry Examples

Every field has a right way and a wrong way to fill it. Below are concrete comparisons.

### Purpose

```
FAIL:
  Purpose: 这一章推进故事发展
  问题：太模糊。"推进故事"是每一章的功能。无法判断这一章的独特价值。

PASS:
  Purpose: Revelation — 揭露盟主与被封印者的兄弟关系，颠覆读者对反派的认知
  为什么好：用了具体的 Purpose 类型 + 一句话说清这一章改变了什么认知。
```

### Tension Level

```
FAIL:
  Tension Level: 7
  （上一章也是7，上上章也是7）
  问题：连续三级相同，违反"无三连同级"规则。读者会感到节奏单调。

PASS:
  Tension Level: 5
  （上一章是9）
  为什么好：在高潮（9）后主动降落到（5），给读者喘息空间。
  对照决策表：上一章 8-10 → 本章应 3-5（recovery）
```

### POV Character

```
FAIL:
  POV: 林北
  （连续15章全是林北POV，其中3章涉及妹妹的关键行动却没切换）
  问题：POV选择服务于便利而非故事。妹妹视角的时刻被跳过了。

PASS:
  POV: 林南（妹妹）—— 本章她在城外被追杀，信息只有她能看到
  为什么好：POV切换有明确的叙事理由。与上一章形成空间对比，增加紧迫感。
```

### Key Events

```
FAIL:
  Key Events: 林北继续调查古剑的秘密，遇到了一些困难
  问题：没有具体事件。"继续调查"是状态不是事件。"一些困难"太模糊。

PASS:
  Key Events:
    1. 林北在图书馆找到封印记录，发现古剑封印的不是魔物
    2. 图书管理员通知他外面有人等着——穿着特殊制服
    3. 他从后窗离开时，听到管理员低声打了一个电话
  为什么好：三条事件都具体、可写、有因果关系。每条事件都是一个场景。
```

### Emotional Target

```
FAIL:
  Emotional Target: 读者会感到紧张和兴奋
  问题："紧张和兴奋"太笼统，几乎所有章节都可以这么说。

PASS:
  Emotional Target: 为主角捏一把汗，但同时对选择感到骄傲——明知危险还要去救
  为什么好：复合情绪（焦虑+骄傲），且和具体叙事内容挂钩。
```

### Continuity Hook

```
FAIL:
  Continuity Hook: 接上一章
  问题：没有说接上一章的什么。写的时候不知道要呼应哪个具体元素。

PASS:
  Continuity Hook: 承接 Ch7 末尾林北决定冒险去救被困矿工——这章他面对决定的后果
  为什么好：精确到章节+具体事件+因果关系。写的时候知道从哪里起笔。
```

### Setup for Next

```
FAIL:
  Setup for Next: 为后面的剧情做铺垫
  问题：哪个剧情？什么铺垫？完全无法验证铺垫是否完成。

PASS:
  Setup for Next: 被封印者说出半句话被中断——下章林北会追查这句话的完整内容
  为什么好：具体的承诺。写下一章时可对照检查：铺垫兑现了吗？
```

## Tension Decision Table

Instead of guessing tension levels, use this decision table based on the previous chapter's tension:

| Previous Chapter | This Chapter Should Be | Rationale | Example |
|-----------------|----------------------|-----------|---------|
| 8-10 (Peak/Climax) | 3-5 (Recovery) | 读者刚经历高潮，需要消化 | Ch28 大战(9) → Ch29 伤员处理(4) |
| 6-7 (Active conflict) | 4-6 (Varies) | 中等张力后可以小幅升降 | Ch15 对峙(7) → Ch16 追击(8)或谈判(5) |
| 4-5 (Building) | 6-8 (Escalate) | 铺垫之后必须推进 | Ch11 情报(4) → Ch12 遭遇敌人(7) |
| 1-3 (Calm/Setup) | 5-7 (Build) | 低谷不能持续，必须在2章内推高 | Ch3 日常(2) → Ch4 发现异常(6) |
| 已连续2章低张力(1-4) | 6+ (Must escalate) | 两章铺垫已到极限 | Ch9(3)+Ch10(4) → Ch11 大事(7+) |
| 已连续2章高张力(7+) | 3-5 (Must breathe) | 两章高压已到极限 | Ch21(8)+Ch22(9) → Ch23 回落(4) |

**How to use:** 填写每章张力值时对照上一章查表。如果表建议与计划冲突——以表为准，或给出充分例外理由。最后做"三连同级"检查（参见 pacing-curve-reference.md 规则1）。

## Tension Curve Application

### Applying the Curve to Your Outline

**Step 1: Set arc-level anchor points first.**
Before filling individual chapters, decide 3-5 anchor chapters and their tension:

```
弧线锚点（第一卷 30章）:
  Ch1: 6 (开篇钩子)
  Ch8: 8 (第一个小高潮)
  Ch22: 9 (中点高潮)
  Ch28: 10 (弧线高潮)
  Ch30: 4 (收束+下弧线钩子)
```

**Step 2: Fill between anchors using the Decision Table.**
Every gap between anchors should have at least one rise and one fall.

**Step 3: Verify with a sequence scan.**
- No three consecutive chapters at the same level
- After every 8-10 there is a 3-5 within 1-2 chapters
- Overall trend points upward (each peak higher than the last)

### Application Example: Filling a 5-Chapter Gap

Given anchor Ch8 (tension 8) and Ch15 (tension 7), fill Ch9-Ch14:

```
Ch8:  8  ← anchor
Ch9:  4  ← decision table: 上一章8-10 → 3-5（recovery）
Ch10: 5  ← building
Ch11: 6  ← 继续 build
Ch12: 7  ← 推向小高峰
Ch13: 5  ← 连续升高后回落
Ch14: 6  ← 向下一个 anchor 靠拢
Ch15: 7  ← anchor
```

Pattern: 8→4→5→6→7→5→6→7。有升有降，无三连同级，整体趋势向上。

## Presenting Outline in Arcs (Not All At Once)

An outline presented arc by arc is manageable and allows organic adjustment. Writing changes plans — building incrementally lets the story evolve.

### Arc Presentation Cycle

**Phase 1: Arc Overview** — Present the arc's overall shape:

```
## 第一卷：觉醒之剑 (Chapter 1-30)

**Arc Theme**: 发现自己并不平凡
**Arc Central Question**: 这把碎剑是什么？为什么选中了我？
**Arc Tension Curve**: [简要描述走向]
**Arc 爽点 Map**: Ch6 小爽, Ch14 中爽, Ch22 大爽, Ch28 大爽
**Arc Start State**: 主角是无名矿工，被追杀，手握神秘碎剑
**Arc End State**: 主角觉醒古剑之力，揭开幕后势力一角，但引发更大危机
```

**Phase 2: First 10 Chapters Detail** — Full chapter entries for the first sub-arc.

**Phase 3: Write, Review, Adjust** — After writing the sub-arc, review and adjust the next batch based on: natural divergence, density issues, new ideas.

**Phase 4: Next Arc Detail** — Present the next batch, incorporating adjustments. Repeat.

## Arc-to-Arc Bridging

### The Bridge Problem

When one arc ends and the next begins, information must carry across. A weak bridge creates narrative whiplash. A strong bridge makes the next arc feel inevitable.

### What Must Carry Between Arcs

Every arc-to-arc transition must explicitly address these six elements:

| Carry Element | What It Is | Where to Record |
|---------------|-----------|-----------------|
| **Unresolved Tension** | 上弧线结束时悬而未决的冲突 | 新弧线 Arc Overview 的"Carried Threads" |
| **Character State Delta** | 角色在上弧线中的变化（成长、创伤、新关系） | 新弧线第一章的 Continuity Hook |
| **Unharvested Foreshadowing** | 已埋但未收回的伏笔（列出ID和状态） | 新弧线的 Foreshadowing Ledger |
| **Open Promises** | 角色做出的承诺或威胁 | 新弧线第一章的 Setup for Next |
| **World Changes** | 上弧线对世界的影响（新势力、新规则） | 新弧线 Arc Overview 的 Start State |
| **Emotional Carry-Over** | 读者在上弧线结束时的情绪状态 | 新弧线第一章的 Emotional Target |

### Bridge Template

At the end of every arc outline, include a Bridge Section:

```
## 弧线桥接：第一卷 → 第二卷

### 承上（Carried Forward）
| ID | 内容 | 状态 | 预计收回 |
|----|------|------|---------|
| F-006 | 林北右手纹路蔓延 | 已埋设，Ch10强化 | 第二卷 Ch3-5 |
| F-012 | 被封印者叫了另一个名字 | 已埋设 | 第二卷高潮 |
| P-001 | 盟主说"我们很快会再谈" | 角色承诺 | 第二卷前10章 |
| T-001 | 被封印者苏醒，下落不明 | 未解决威胁 | 第二卷主线 |

### 启下（New Arc Seeds）
- 第二卷 Central Question: 被封印者是谁？他叫的那个名字是什么？
- 第二卷开篇情绪: 不安+好奇（承接"封印破裂的隐忧"）
- 第二卷第一章开头3段内必须提及至少一个 Carried Forward 元素

### 角色状态传递
| 角色 | 第一卷结束状态 | 第二卷起点状态 |
|------|-------------|-------------|
| 林北 | 觉醒古剑之力，右手异常 | 力量不稳定，纹路扩散 |
| 林南 | 被救出，目睹哥哥异常 | 距离感+担忧 |
```

### Bridge Density Guidelines

- **紧接式弧线**（上弧线高潮直接引发下弧线）：新弧线第一章张力值应 6-7，保持惯性
- **间歇式弧线**（两弧线之间有时间跳跃）：新弧线可从 3-4 起步，但必须在第一章内激活一个 carried thread
- 新弧线的前三章必须至少收回一条 carried foreshadowing，否则读者会觉得伏笔被遗忘

## Foreshadowing Planning: Seeds and Harvests

**伏笔 Card Format:**

```
[FS-001-01] 古剑的呼唤
- 埋设章节: Ch1  |  方式: 主角感觉古剑在"呼唤他"
- 强化章节: Ch8, Ch15（微弱提示，加深记忆）
- 收回章节: Ch45 |  方式: 揭示呼唤来自被封印者，非剑本身意志
- 故事意义: 颠覆对古剑性质的认知
```

### Foreshadowing Rules

1. **Plant-Harvest Distance**: Major foreshadowing 5-10 chapters before harvest. Minor: 2-3 chapters.
2. **The 3-Touch Rule**: Plant (subtle) → Reinforce (remind without explaining) → Harvest (payoff).
3. **Plant Naturally Through Action**: Vary planting methods — action, dialogue, environment. "他注意到那个奇怪的符号但没有在意" is a red flag.
4. **Some Seeds Don't Sprout**: Include world-texture seeds that seem like foreshadowing but aren't. This masks the real payoffs.
5. **Harvest Must Exceed Expectation**: Payoff should be more interesting than what the reader guessed.

## 网文-Specific: Mini-Arc Structure

A mini-arc is a 5-10 chapter self-contained unit with its own Setup, Rising action, Climax, Resolution (partial), and Hook to next mini-arc. 网文 readers read in sessions of 5-10 chapters — each session should feel satisfying on its own.

### Mini-Arc Template

```
## Mini-Arc: [名称] (Chapter X - Chapter Y)
**Purpose**: [在整体故事中的作用]
**Conflict**: [核心冲突]
**爽点 Type**: [爽点类型]
Ch X:   Setup — 引入冲突
Ch X+1: Build — 冲突升级
Ch X+2: Complication — 意外因素
Ch X+3: Climax — 小弧线高潮/爽点
Ch X+4: Aftermath — 后果+下个弧线钩子
```

### 爽点 Density Planning

| Story Phase | Major (大爽) | Minor (小爽) | Interval |
|-------------|-------------|-------------|----------|
| Opening (Ch1-10) | 1 | 2-3 | 大爽 every 5-8 ch |
| Rising (Ch11-40) | 2-3 | 4-6 | 大爽 every 8-12 ch |
| Midpoint (Ch41-60) | 1-2 | 3-5 | 大爽 every 8-10 ch |
| Climax Build (Ch61-85) | 2-3 | 4-6 | 大爽 every 6-10 ch |
| Climax+Resolution (Ch86-100) | 2 | 3-4 | 密集区，更频繁 |

Vary 爽点 spacing deliberately: no more than 8 chapters without at least a minor 爽点. After a major 爽点, allow 2-3 chapters of processing. The biggest 爽点 should come after the longest build-up.

## Outline Smoke Test

After filling an arc outline, run this checklist BEFORE writing. Every check must have a concrete answer — "looks good" is not an answer.

### 1. Tension Sequence Scan

- [ ] **三连同级**: 连续3章以上相同张力值的位置存在吗？如果存在，调整。
- [ ] **高潮-回落**: 每个 8+ 张力章后面，1-2章内有 3-5 的回落吗？
- [ ] **低谷-攀升**: 每个连续2章的低谷(1-4)后面，下一章攀升到 5+ 了吗？
- [ ] **整体趋势**: 弧线后半段平均张力 > 前半段？
- [ ] **最高点位置**: 全弧线最高张力值落在 70-90% 位置吗？

### 2. Chapter Integrity Check

- [ ] **可删除测试**: 删除这一章读者会丢失什么？答案是"没什么"则该章无存在理由。
- [ ] **Purpose 唯一性**: 连续三章同一 Purpose 是结构问题。
- [ ] **Emotional Target 区分度**: 弧线中所有 Emotional Target 列出，有两条几乎一样的就需调整。

### 3. Continuity Chain Check

- [ ] **前后匹配**: Ch(N) "Setup for Next" 与 Ch(N+1) "Continuity Hook" 形成问答对？逐一对照。
- [ ] **Foreshadowing 完整性**: 每个伏笔ID都有 plant + scheduled harvest。未收回的标记"延迟"或"移至下弧线"。

### 4. 网文 Health Check (if applicable)

- [ ] **前三章 Hook**: Ch1 开头 500 字内建立了让读者想追下去的问题？
- [ ] **爽点间距**: 最长无爽点区间 ≤ 8章？
- [ ] **章末钩子**: 每章结尾有让读者想继续读的元素？
- [ ] **连续铺陈上限**: 无超过5章连续 setup/exposition 无爽点。

### 5. Bridge Readiness Check (if not the last arc)

- [ ] **Carry Element 清单**: 列出了所有需要传递到下一弧线的元素？
- [ ] **最后三章弧线钩子**: 至少一章的"Setup for Next"指向弧线外？
- [ ] **Emotional Carry-Over**: 最后一章的 Emotional Target 能为新弧线开篇提供情绪起点？

## Example: Full Mini-Arc Outline

```
## Mini-Arc: 矿坑危机 (Chapter 6-10)
**Purpose**: 第一个战斗弧线，展示主角在压力下的选择
**Conflict**: 矿坑深处异常魔兽，矿工被困
**爽点 Type**: 守护成功型 + 力量初现型

### Chapter 6: 异变
| **Purpose** | Setup — 引入威胁 |
```

## Outline as Living Document

The outline is NOT set in stone. It evolves with the writing.

### When to Revise
- After each mini-arc, review the next mini-arc's outline
- If the story diverges naturally, adjust rather than force it back
- If a scene is richer than expected, let it breathe (split into two chapters)
- If a scene falls flat, diagnose and fix the outline before writing around it

### Outline Revision Dialogue
- "原计划是[X]，故事走向了[Y]。矛盾吗？不矛盾就调整后续。矛盾就聊哪个更好。"
- "这个场景写得顺畅吗？顺畅说明大纲方向对。勉强说明大纲在强行推故事走它不想走的路。"

The outline serves the STORY, not the other way around. If the story wants to go somewhere the outline didn't plan, listen to the story first.
