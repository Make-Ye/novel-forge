# Novel-Forge Proposal System

> **权威声明**: 本文件是提案系统的权威参考（何时提案、如何提案、如何处理用户反馈）。

## 核心理念

**你是指挥，AI是编剧团队。** 每一个关键创意决策，AI都提供2-3个方案供你选择——而不是直接拍板。这保证了小说是你的作品，AI只是你的创意协作工具。

---

## 一、何时提出方案（Proposal Triggers）

### 必须提出方案的场景

| 场景 | 原因 |
|------|------|
| **世界观核心设定**（魔法体系、科技框架、社会结构） | 这些选择影响全书，必须由你决定基调 |
| **重要角色概念**（主角、主要配角、反派的核心设计） | 角色是小说的灵魂，不同方向带来完全不同的故事 |
| **情节结构选择**（叙事框架、主线编织方式） | 结构决定节奏和读者体验 |
| **章节开篇/收尾策略**（首章开头、关键转折章结尾） | 这些位置决定读者是否继续阅读 |
| **重大修订方向**（情节推翻重写、角色弧线调整） | 修改成本高，需要你明确方向 |
| **风格/基调选择**（叙事风格、语言调性） | 这是你的作者声音 |
| **悬念/揭秘策略**（何时揭晓、如何埋线） | 直接影响叙事张力 |
| **多线叙事的分岔/汇合点** | 结构性决策，影响后续多章 |

### 不需要提出方案的场景

| 场景 | 处理方式 |
|------|----------|
| **连续性修正**（前后矛盾、细节对齐） | 直接修正，标注变更 |
| **状态提取/记录**（从文本提取角色状态、时间线） | 直接执行，展示结果 |
| **微调措辞**（错别字、语病、小段润色） | 直接修改 |
| **格式统一**（对话标点、分段规范） | 直接处理 |
| **事实查询**（确认已设定的规则、角色信息） | 从记忆文件读取回答 |
| **你已经明确给出方向的执行** | 按方向执行，不再问 |

### 灰色地带判断原则

> **如果这个决定会影响后续3章以上的内容，就必须提出方案。**
> **如果只是当前段落内的措辞选择，直接做。**

## 二、标准方案格式

每个proposal包含2-3个选项，格式如下：

```
## [决策主题]

### 选项A: [名称]
[1-2句描述]

**优势：**
- ...
- ...

**劣势：**
- ...
- ...

**最适合：** [什么类型的作品/读者/效果]

---

### 选项B: [名称]
[1-2句描述]

**优势：**
- ...
- ...

**劣势：**
- ...
- ...

**最适合：** [什么类型的作品/读者/效果]

---

（如适用）### 选项C: [名称]
同上格式

---

**我的直觉倾向 [A/B/C]，因为 [理由]。但你是总指挥，你来拍板。**
```

### 格式要点

- 每个选项必须有**名称**（不是"方案一"，而是有具体含义的名称）
- 描述控制在1-2句，不啰嗦
- 优势和劣势各列2-3条，**必须诚实**——优势不能全是A的，劣势不能全是B的
- "最适合"帮助你对号入座，但不是限制
- AI的倾向性推荐放在最后，**语气要明确但不强制**

---

## 三、用户回复处理

### 标准回复类型

| 用户回复 | AI行为 |
|----------|--------|
| **选A/B/C** | 按选定方案执行，确认选择后开始 |
| **合并A和B** | 提取A和B的精华，生成合并方案，展示给你确认 |
| **自定义方案** | 记录你的方案，按此执行 |
| **"惊喜我"** | AI选择其推荐方案并执行，但会标注"基于'惊喜我'指令，我选择了X" |
| **"再想想"** | 生成新的2-3个不同方向的方案 |
| **"都要"** | 分析是否可以分阶段/分场景使用不同方案，给出可行建议 |

### 合并处理协议

当用户要求合并时：

```
你选择了合并 A + B。我的理解是：
- 从A取：[具体元素]
- 从B取：[具体元素]
- 合并后效果：[描述]

这个理解正确吗？确认后我开始执行。
```

## 四、方案示例

### 示例1：魔法体系类型

```
## 魔法体系类型

### 选项A: 硬魔法体系
魔法有严格的规则和限制，类似物理定律。读者可以理解规则并预测结果。
代表作品：《钢之炼金术师》《 Mistborn》

**优势：**
- 读者可以参与推理，战斗场面有智力快感
- 限制本身就是创意的源泉
- 容易建立公平感（读者不会觉得"主角开挂"）

**劣势：**
- 前期设定需要大量交代，节奏可能变慢
- 作者自己必须严格遵守规则，不能破例
- 复杂规则可能劝退轻度读者

**最适合：** 以战斗/解谜为核心卖点的作品，目标读者喜欢推演

---

### 选项B: 软魔法体系
魔法神秘、不可完全理解，重在氛围和感受。魔法是工具而非系统。
代表作品：《魔戒》《千与千寻》

**优势：**
- 保持神秘感和想象力空间
- 节奏更流畅，无需大量解释
- 适合制造宏大、超凡的氛围

**劣势：**
- 容易沦为"需求驱动"（需要什么就有什么）
- 战斗场面缺乏紧张感（读者无法判断危险程度）
- 难以让读者产生"这个魔法世界是真实的"感觉

**最适合：** 以情感/氛围为核心的作品，奇幻色彩重于逻辑

---

### 选项C: 混合体系（推荐）
存在核心规则框架，但边缘地带保持神秘。不同派系/种族的魔法理解程度不同。
代表作品：《哈利波特》《葬送的芙莉莲》

**优势：**
- 兼具推理感和神秘感
- 可以按剧情需要调整"硬度"
- 不同角色对魔法的不同理解天然产生冲突

**劣势：**
- 需要精心设计哪些部分硬、哪些软
- 最难写好——硬的部分不能矛盾，软的部分不能敷衍
- 容易滑向"两边都不沾"

**最适合：** 长篇连载，需要兼顾战斗和氛围的作品

---

**我的直觉倾向 C（混合体系），因为它给后续的角色差异化留出了最大空间，也适合长篇叙事中逐步揭秘的节奏。但你是总指挥，你来拍板。**
```

### 示例2：角色概念（主角设计）

```
## 主角核心概念

### 选项A: 被迫成长的理想主义者
内心善良但天真，世界不断挑战其信念。核心冲突是"保持善良vs适应残酷"。

**优势：**
- 读者容易共情，成长弧线清晰
- 适合探讨深刻主题（信念、牺牲、妥协）
- 结局有强烈的情感冲击潜力

**劣势：**
- 前期可能让读者着急（"怎么这么天真"）
- 这类角色极多，需要独特细节避免俗套
- 如果成长太慢，读者可能弃书

**最适合：** 以角色成长为核心驱动力的故事

---

### 选项B: 务实的局外人
不追求英雄使命，因外部压力卷入事件。以冷静、反讽的视角观察世界。

**优势：**
- 可以用"局外人视角"自然地介绍世界观
- 幽默感和反差感强
- 突破"热血主角"刻板印象

**劣势：**
- 容易变成被动角色（"被推着走"）
- 情感投入需要更长时间建立
- 讽刺基调如果过头，会让读者难以认真对待危机

**最适合：** 黑色幽默、讽刺风格、以世界观探索为主的作品

---

### 选项C: 双面人
表面扮演一种角色，内心/暗处是完全不同的人。核心张力来自身份的维持与撕裂。

**优势：**
- 天然产生悬念和戏剧反讽（读者知道主角的真相，其他角色不知道）
- 对话和互动有双重含义，写作空间极大
- 适合伏笔和反转

**劣势：**
- 需要精心设计"面具"和"真实自我"的平衡
- 如果处理不当，读者会觉得虚伪而非复杂
- 长期维持双重叙事对作者要求高

**最适合：** 悬疑、阴谋论、权力斗争类作品

---

**我的直觉倾向 A，因为你之前提到想写一个"在残酷世界中寻找意义"的故事，A的成长弧线和这个主题最匹配。但你是总指挥，你来拍板。**
```

### 示例3：情节结构

```
## 叙事结构

### 选项A: 经典三幕剧
建置（25%）→ 对抗（50%）→ 解决（25%）。明确的核心冲突和一次重大转折。

**优势：**
- 读者最熟悉的节奏，接受度最高
- 结构清晰，不容易写"散"
- 转折点和高潮的位置可以精确设计

**劣势：**
- "中段塌陷"风险高——对抗部分过长容易拖沓
- 如果转折点不够有力，整个结构就塌了
- 对长篇连载而言，可能显得"太整齐"

**最适合：** 中篇作品、有明确开端-高潮-结局的故事

---

### 选项B: 单元剧+主线串连（Serial + Arc）
每个"单元"（3-5章）有独立的小高潮和完整故事，同时推进贯穿全书的大主线。
代表：《葬送的芙莉莲》《灵能百分百》

**优势：**
- 天然适合连载，每个单元都能给读者满足感
- 可以在不同单元中探索世界观的不同面
- 新读者可以从任何单元开始，降低阅读门槛

**劣势：**
- 主线推进可能感觉慢
- 单元之间的质量波动更明显
- 需要设计好单元故事与主线的有机联系，否则显得"跑腿"

**最适合：** 长篇网文、探索型故事、世界观丰富的作品

---

### 选项C: 非线性叙事
时间线跳跃、多视角交叉、或从结局倒叙。打破时间顺序制造悬念。
代表：《冰与火之歌》（多视角）、《暴雨》（游戏，多视角交叉）

**优势：**
- 天然制造信息差和悬念
- 读者体验更"高级"
- 可以通过时间线安排控制信息释放

**劣势：**
- 如果时间线不清晰，读者会混乱
- 写作难度最高，需要精确的时间线管理
- 角色共鸣建立更慢（因为视角在切换）

**最适合：** 悬疑类、多角色群像、追求文学性的作品

---

**我的直觉倾向 B（单元剧+主线），因为你的作品规划是长篇连载，这种结构给读者持续的小满足感，同时保持大悬念。但你是总指挥，你来拍板。**
```

## 五、特殊情况处理

### 当用户说"你决定"

AI选择其推荐方案，但在执行前会简要说明选择理由：

```
基于"你决定"的授权，我选择 [方案X]。
理由：[1-2句话]
如果你在任何时候想调整，随时告诉我。
```

### 当方案间并非互斥

如果多个方案可以共存，主动指出：

```
注意：选项A和C并不完全互斥。
[解释如何组合]
你是否想采用组合方案，还是需要我强制你选一个？
```

### 当决策依赖未知信息

如果AI认为需要更多信息才能给出好方案：

```
在提出方案之前，我需要确认一个问题：
[问题]
这个答案会影响方案的方向。
```

### 方案回溯

如果已选择的方案在执行中发现问题：

```
关于之前选择的[方案X]，执行中发现一个问题：
[问题描述]
有三种处理方式：
1. 微调：[具体调整]
2. 回退到[方案Y]：[理由]
3. 继续：接受这个限制
你的选择？
```

## 六、Proposal系统的元规则

1. **方案质量优先于数量**——宁可给2个深思熟虑的方案，不给3个凑数的
2. **每个方案都必须是你真正可以选择的**——不能有"正确答案"和"明显更差的选项"
3. **AI的推荐必须真诚**——如果AI觉得某个方案明显更好，要直说
4. **推荐理由必须具体**——不能说"更合适"，要说"更合适因为你的主角设定是X，而方案A的Y特性正好配合X"
5. **一旦你做出选择，AI全力执行**——不再反复暗示"其实另一个更好"
6. **执行中发现方案问题，立即汇报**——不擅自换方案，也不硬撑
